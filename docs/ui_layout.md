# Moon Spire UI 布局与 PNG 资源说明

Moon Spire 的自定义界面底图都放在：

`src/main/resources/assets/moonspire/textures/gui/ui/`

战斗效果图标等状态类 PNG 单独放在：

`src/main/resources/assets/moonspire/textures/gui/effects/`

战斗过程中的一次性动画 PNG 单独放在：

`src/main/resources/assets/moonspire/textures/gui/animations/`

这些 PNG 是当前仍有有效代码路径的模组重绘材质入口。未被 UI 代码引用的旧贴图会从资源目录移除；后续需要恢复某个独立视觉样式时，应先重新接入渲染代码，再加入对应 PNG。卡牌本体底图仍在：

`src/main/resources/assets/moonspire/textures/gui/cards/card_base.png`

## 布局文件

`src/main/resources/assets/moonspire/ui/battle_layout.json`

开发调试覆盖文件：

`moonspire/developer/ui/battle_layout_override.json`

旧版兼容读取来源：

`config/moonspire/battle_layout_override.json`

在 Gradle 开发运行目录中，客户端配置和覆盖文件通常位于：

`run/config/moonspire/client.json`

`run/moonspire/developer/ui/battle_layout_override.json`

覆盖文件和默认布局使用同一套 schema。调试完成后，可以把覆盖文件中的 `elements` 值复制回默认资源 JSON，用作要随模组发布的布局。

当前默认战斗布局已同步自用户运行时覆盖文件：

`D:\Minecraft\PCL启动器\.minecraft\versions\月亮尖塔\moonspire\developer\ui\battle_layout_override.json`

发布布局采用顶部条目贴近屏幕边缘、怪物意图位于 `top_center y:1 scale:0.74999994`、抽牌堆和弃牌堆贴底角、费用球使用 `scale: 0.5999999`、结束回合按钮位于右侧中部且宽度为 68，按钮本体只显示回合文案，快捷键提示写在悬停提示框中、手牌区域位于 `bottom_center y:-21`、出牌区域位于 `center x:0 y:-23 width:512 height:230 scale:1.2499998` 的调试结果。

## 开发者模式

创建或编辑：

`config/moonspire/client.json`

内容示例：

```json
{
  "developerMode": true
}
```

`key.moonspire.ui_debug` 默认绑定为 F8，但只有客户端启动时 `config/moonspire/client.json` 已开启 `developerMode` 才会注册到 Moon Spire 按键分类中；关闭开发者模式时，Controls 中不会显示开发者中心/调试入口，也不会留下问号占位项。按下该键时仍会重新读取 `developerMode` 作为二次保护；若此时开发者模式已关闭，F8 会静默无反应，不显示提示、不打开界面、不向服务器发包。若从关闭改为开启，需要重启客户端后才会在按键控制中出现 F8 入口。编辑器仍然只能在战斗进行中打开；普通玩家未开启开发者模式时不会看到布局编辑器。

战斗界面只会在没有其它 Screen 打开时由客户端自动补回。暂停菜单、退出世界、断开连接等原版界面流程不应被战斗界面覆盖；客户端退出或世界为空时必须清空本地战斗状态和布局编辑器状态。

战斗界面本体不绘制暗色透明背景，必须保持世界画面直接可见。只有卡组查看、牌堆弹窗、布局编辑器等覆盖层可以使用 `panel_dark.png` 或其它暗色透明底；整屏遮罩由代码直接绘制，不再依赖独立 `overlay.png`。

## 布局字段

每个布局元素包含：

- `anchor`：锚点，可用 `top_left`、`top_right`、`top_center`、`bottom_left`、`bottom_right`、`bottom_center`、`center`、`mouse`。
- `x`、`y`：相对锚点的偏移。
- `width`、`height`：元素基础尺寸。
- `scale`：最终缩放倍率。

稳定战斗布局元素 id：

`player_entry`、`monster_entry`、`monster_intent`、`energy`、`draw_pile`、`discard_pile`、`exhaust_pile`、`end_turn`、`hand`、`play_area`、`card_preview`、`tooltip`。

旧版调试元素 `pile_overlay`、`world_bar` 和 `world_effects` 已移出布局 schema。加载旧覆盖文件时，代码会忽略这些无效 id，避免它们重新出现在布局编辑器中；世界头顶血条和效果仍由 `BattleWorldOverlay` 在实体 billboard 坐标内固定绘制，不属于屏幕 UI 布局编辑范围。

## 编辑器操作

- 点击 UI 元素选中；当前选中 UI 的黄色轮廓会最后绘制，避免被其它 UI 轮廓覆盖。
- Tab 选择下一个 UI 元素。
- Shift + Tab 选择上一个 UI 元素。
- 拖动选中元素调整位置。
- 滚轮调整缩放。
- Shift + 滚轮调整宽度。
- Ctrl + 滚轮调整高度。
- 方向键移动 1 像素。
- Shift + 方向键移动 10 像素。
- Ctrl + 方向键调整宽度或高度。
- Ctrl + S 保存覆盖文件。
- Ctrl + R 重新加载布局数据。
- Ctrl + D 重置为默认资源布局。

## 覆盖层输入与背景

- 战斗内的抽牌堆、弃牌堆和卡组查看覆盖层打开时，覆盖层本身负责吞掉鼠标点击、拖拽、滚轮和键盘输入，并清空底层战斗 hover、拖牌、相机拖动和待选目标状态；右键或 ESC 只关闭当前覆盖层。抽牌堆和弃牌堆的 HUD 提示框必须避开被指向的牌堆图标矩形，不能遮挡玩家正在查看或点击的牌堆入口。
- 卡牌网格覆盖层的暗色背景由 `MoonSpireUiTextures.drawOverlay()` 直接绘制为箱子式暗色透明遮罩，不依赖 PNG 像素透明度，因此不能呈现为完全不透明的纯黑背景。该 helper 必须在绘制前提交底层 GUI，再绘制遮罩并立刻提交遮罩本身，随后才允许 modal 标题、按钮、卡牌和提示绘制到遮罩上方。
- 战斗内牌堆覆盖层不再使用可调布局元素，而是由 `CardGridPanel` 按当前屏幕宽度和底部预留直接铺开；`DeckScreen` 是纯卡牌查看界面，不创建底部按钮，也不再通过 bottom reserve 为按钮保留空间。
- `play_area` 控制非指向目标卡牌的有效出牌区域。玩家拖动格挡、自身收益、全体或随机目标等不需要箭头精确命中的卡牌时，鼠标进入该矩形才会显示可打出反馈并允许释放；区域底部仍会被手牌区域顶部略微截断，避免在手牌上误触发出牌。
- 怪物意图小卡和放大预览卡牌本体会阻止底层怪物目标选择与 hover 更新，避免查看意图牌时选中怪物或造成预览闪烁；关键词提示区不参与 hover 保持判定。
- 战斗中的 hover/拖牌高亮会同时作用于条目和世界实体。玩家不会被点击选中，但会在鼠标指向或自我收益牌进入有效出牌区域时显示蓝色轮廓；怪物仍可被点击选中并显示黄色轮廓。若卡牌效果里包含单体敌方或单体友方，就必须拖动箭头实际指向对应目标，并且拖拽高亮只保留当前指向的单体目标，不能同时保留已选中的第一只怪物。全体友方、自身以外的所有单位等范围目标不要求拖箭头。单体友方不包含自己，当前 1v1 战斗里没有其它友方时不会把玩家当作合法单体友方目标。如果同一张牌有多条效果，则进入有效出牌区域时按实际会高亮的目标数量选择“覆盖单位数量最高”的目标组，当前以单体、自身、全体敌方、全体友方、全体单位、自身以外的所有单位和随机敌方目标为准。条目区滚动裁剪必须给蓝色 hover 框和黄色选中框保留边缘像素，不得切掉第一行或最后一行的外框。
- 怪物正在使用的牌绘制在 `monster_intent` 布局区域中心，尺寸只比意图小卡略大，且会在屏幕内自动夹紧，避免战斗窗口较小时卡面越界；该牌仍然是最顶层表现层，不会被其它卡牌遮住。

## 滚动条规则

- 所有带滚动条的 Moon Spire UI 都只在内容无法完整显示时绘制并允许滚动条滚动。
- 所有可能出现多内容且显示不全的 Moon Spire UI 区域都必须支持滚动；内容溢出时至少要支持鼠标滚轮滚动，并在使用滚动条呈现时支持滚动条拖动。
- 所有多内容区域都必须为内容的上、下、左、右完整可见预留内边距，并把该内边距纳入内容高度、裁剪区、滚动上限和点击命中计算；不能让第一行或最后一行卡牌、图标、文字、选中边框被裁剪区切掉。
- 滚动必须是连续像素滚动，不能按页或按整行跳转。
- 滚动条滑块必须支持直接拖拽。
- 滚动后不能跳过某一行或让某一行永久看不见。
- 内容能完整显示时不显示滚动条，滚轮不会移动内容；内容不能完整显示时，滚动位置直接限制在顶部到底部的合法范围内，不做越界或回弹。
- `CardGridPanel` 负责卡组、抽牌堆、弃牌堆等卡牌网格滚动；网格优先按 5 列显示，并会缩小网格卡牌以保证常规卡组宽度下一行能容纳 5 张。卡牌悬停放大以原卡牌中心为锚点绘制，并跳过原位小卡，避免叠加造成额外发光；离开悬停后会平滑缩回原位，不会瞬间消失。网格裁剪区从牌组标题和数量下方开始，只保留标题区本身作为顶部遮挡；底部不再为了按钮或边缘安全距额外压缩可视高度，普通卡牌可绘制到屏幕底部。滚动内容顶部和底部会加入只作为空位的额外滚动距离，不作为遮盖层，使第一行和最后一行卡牌滚动后悬停放大也能完整显示。滚动时已放大的卡牌会直接贴住当前原卡牌中心移动，不产生位置追赶延迟。放大卡牌本体不再被牌堆/卡组可用边界夹动，也不会被强行移到屏幕中间。关键词提示会优先显示在卡牌右侧，右侧空间不足时显示在左侧；提示整体尽量相对卡牌高度居中，只有超过卡牌顶/底或屏幕边缘时才向下/上修正，不能反向挤动放大卡牌本体。
- `CardForgeScreen` 的可转换物品列表也使用相同的滚动条规则。
- `CardForgeScreen` 的可转换物品条目在鼠标指向时会在 `forge_slot.png` 槽位上方绘制亮色底和金色边框作为高亮反馈。指向某个可转换条目时，界面右侧会用完整卡牌尺寸绘制该物品即将转换成的卡牌详情，并沿用 `CardRenderHelper.renderKeywordTipsBeside(...)` 在卡牌左右侧显示关键词说明；这只是预览，不改变点击条目后消耗物品并生成卡牌的规则。右侧预览不新增 PNG 资源，继续复用卡牌底图、物品图标和关键词提示框。

## 战斗卡牌预览

- 战斗手牌按底部浅弧形扇排显示，卡牌之间保持明显重叠；中间卡牌只略高更正，两侧卡牌轻微下沉和旋转，少量手牌必须保持平缓，不能形成尖弧或明显高拱。
- 手牌悬停预览由 `BattleScreen` 的 `HandCardAnimation` 缓存驱动，从原手牌位置平滑上抬、回正并克制放大，原小卡不再同时绘制；鼠标离开后平滑回位。动画差分由 `ClientBattleState` 的快照版本触发，只在战斗快照变化时同步；稳定渲染帧按真实帧间隔推进手牌与飞牌插值，并读取帧内缓存的手牌布局、hover 和目标命中，避免卡牌动画被 20 TPS 游戏 tick 限制，也避免重复布局和目标重写造成卡顿。开发者模式下会定期输出战斗卡牌动画的轻量性能统计。
- 拖拽手牌时，从原小卡或放大预览卡牌本体按下都可以开始拖拽。技能牌和自我目标牌会跟随鼠标并使用与手牌放大预览一致的详细卡牌表现；攻击牌始终保留从卡牌指向鼠标的瞄准箭头，不会在命中怪物时把整张牌移动到目标位置。需要目标的攻击牌可以拖到怪物基础条目上直接打出，也可以在世界中直接指向怪物打出；拖拽到可打出位置时，卡牌会显示可打出背景高亮；同一次拖拽中只要卡牌离开可打出区域后再次进入，就会再次触发更强的短促高亮。
- `BattleScreen` 通过手牌 UUID、牌堆快照和 phase 对比快照，推断抽牌飞入、非出牌手牌离场飞向弃牌堆，以及玩家/敌方回合切换提示。玩家和怪物打出的牌都会先以普通手牌小卡大小进入战场展示位，并在服务端效果结算期间保持在屏幕中；结算结束并进入目标牌堆后，普通牌再缩小飞向弃牌堆，带“消耗”的牌必须停在战场展示位原地淡出，不能飞向弃牌堆。打出后的卡牌不再按当前费用显示费用不足，因为它已经不是手牌。这些都是客户端表现层，不改变服务端结算协议。
- 可打出的手牌会显示柔和青色高亮；费用不足或不可打出的手牌不额外变暗，只是不显示可打出高亮；卡牌费用只显示单层描边数字，字号以正常手牌小卡为基准，并随放大预览、卡组和牌堆中的卡牌实际显示比例同步缩放；费用不足时卡牌费用数字显示为红色，放大预览也必须同步变红。战斗当前费用使用费用图标加当前/最大费用比值显示，0 费使用灰色图标，有费用时使用金色图标。结束回合按钮文字使用单层描边、不带阴影；按钮内只显示四类文案：本地玩家可结束回合时显示“结束第%s回合”，怪物回合时显示“怪物回合”，本地玩家假死亡时显示“你已死亡。”，多人战斗中本地玩家已结束回合且仍需等待队友时显示“等待其他玩家结束回合”。顶部回合横幅只显示“玩家回合”和“怪物回合”，不显示“回合结束”；“玩家回合”等阶段提示不放进结束回合按钮。可结束且鼠标悬停时为红字，没有可出牌时按钮保持蓝色高亮并显示黄色文字，但按钮底图仍要根据鼠标悬停状态切换；长文案都必须在按钮内部自动缩小并保持水平、垂直居中。
- 战斗 HUD 的速度、费用、结束回合、抽牌堆和弃牌堆都会在鼠标指向时显示 `tooltip.png` 九宫格提示框，并在最顶层绘制。速度提示只绑定条目中实际绘制的速度文字矩形，矩形由当前 `player_entry` / `monster_entry` 布局位置和速度文字宽度实时计算，不能使用固定屏幕坐标；费用、结束回合、抽牌堆和弃牌堆提示分别使用对应布局元素命中范围。结束回合按钮文字追加快捷键括号，并在按钮宽度不足时等比缩小但保持居中。
- 怪物意图牌悬停预览以原小卡中心为基准原地放大，并使用与卡组/牌堆网格一致的平滑放大表现；只有碰到屏幕边缘时才轻微推回屏内。预览本体可保持悬停，旁边关键词提示不会扩大 hover 区域；关键词提示在左右侧选边后尽量相对卡牌高度居中，只有越出屏幕顶/底时才做最小上下修正。拖拽需要目标的攻击牌并指向怪物时，不弹出怪物意图牌预览，避免压住出牌瞄准反馈。
- 战斗选中只表示怪物意图查看：只有怪物条目和世界怪物能被点击选中并显示黄色轮廓，玩家不能被选中。鼠标 hover、拖牌目标和自我目标牌指示使用蓝色轮廓，可作用于玩家或怪物，并且条目和世界高亮必须同步。攻击牌不再使用已选中怪物作为释放回退，必须拖动箭头实际指向怪物条目或世界怪物才能打出；格挡等自我目标牌进入有效出牌区域时高亮玩家。
- 战斗外准星指向 10 格内可挑战生物时，`BattleHud` 会在准星旁绘制“按[挑战目标键]挑战目标”提示；按键名称读取当前按键绑定。该提示不使用新的 PNG 资源，只作为 HUD 文本绘制；战斗界面激活时不显示。提示和挑战键都使用 `ClientEvents.challengeTarget()` 的 10 格射线检测，不能依赖原版较短交互距离的 `minecraft.hitResult`。
- 怪物正在使用的牌绘制在 `monster_intent` 布局区域中心，使用比意图小卡略大的卡牌尺寸，并在基础战斗 UI、怪物意图小卡、手牌、飞牌和拖拽牌之后绘制，保证它是战斗卡牌层的最顶层。新怪物牌出现时立即替换旧展示，最后一张牌结算完成后保留约 1 秒再移除；该计时按渲染帧真实时间推进。
- 卡牌描述中的关键词统一显示为与侧边关键词详情标题一致的较深金黄色，包括格挡、流血和守护等效果名称；卡牌描述正文使用与关键词详细描述相同的浅白色。卡牌描述会随目标变化：单体敌方和自身保留简洁写法，其它目标会明确写出“对全体敌方”“给予随机敌方”等前缀。手牌、拖拽卡、怪物意图牌、卡组/抽牌堆/弃牌堆中的放大预览都会在卡牌左侧或右侧展示对应关键词详情，并按屏幕剩余空间选择展示侧。关键词和效果说明提示框必须作为最高层说明绘制，不被放大卡牌、开发者界面控件或其它普通 UI 内容压住。卡牌数值高于原值时的绿色提示使用更深的绿色，避免过亮。
- 基础条目和世界头顶条的生命/格挡使用同一套 `CardRenderHelper.combatantBarSegments()` 分段规则：先绘制底槽，再用 `health_fill.png` 绘制当前生命，用 `block_fill.png` 从生命段右侧接续绘制格挡段，生命和格挡按当前生命 + 当前格挡的有效耐久比例并排分布，避免两层填充互相覆盖或错位。世界头顶条复用 `health_bar.png` 作为底槽，底槽、生命填充和格挡填充使用固定内框尺寸；内框按 `health_bar.png` 暗槽使用 2 像素边距，填充贴图使用完整 UV，使可见色块贴合底槽暗色内框，并用轻微 Z 分层避免同平面叠加闪烁。世界头顶 UI 整体锚定在实体高度上方并额外上移，确保第一行战斗状态、血条和效果不会卡进生物头部或燃烧遮挡里。基础条目只保留名称、速度、血条、效果和轮廓；鼠标瞄准/悬停为蓝色边框，只有选中目标为黄色边框。
- 多人多怪战斗中，`player_entry` 和 `monster_entry` 布局元素作为双方条目的起始位置；同阵营多个条目按条目高度加 8 像素间距向下堆叠。世界头顶血条、高亮轮廓和假死亡动画会遍历所有参战单位，而不是只处理本地玩家和第一只怪物。

## 开发者中心 UI

- `key.moonspire.ui_debug` 默认仍为 F8，但只有 `config/moonspire/client.json` 开启 `developerMode` 时才注册并显示在按键控制中；它现在优先作为开发者中心入口，进入时还需要世界内拥有 2 级管理员权限。
- 战斗进行中按 F8 仍会切换战斗 UI 布局编辑器，编辑器覆盖文件保存到版本目录 `moonspire/developer/ui/battle_layout_override.json`；旧的 `config/moonspire/battle_layout_override.json` 仍会作为兼容读取来源。
- 开发者中心的删除确认、效果选择等 modal/弹窗必须使用统一的顶层覆盖方法：先绘制底层内容；若 modal 会覆盖底层 widget 文字，例如删除确认，则不要继续绘制底层 `renderWidgets(...)`，避免页签、按钮和输入框文字在弹窗背后保持高亮；随后绘制 `MoonSpireModalLayer.drawTopmostOverlay(...)` 的箱子式半透明暗遮罩，最后绘制 modal 本体。不要用近乎不透明的整屏黑层来掩盖底层文字问题；正确做法是让被 modal 阻断的底层 widget 不参与绘制。modal 关闭必须通过 `MoonSpireModalLayer.close(...)` 或同等的“清理交互状态后重建控件”流程，避免取消目标后底层按钮、卡牌、滚动条或拖拽状态残留高亮。
- 开发者中心包含卡牌、卡面、怪物和战斗 UI 页面。卡牌页左侧为卡牌名称、费用、卡牌图片和卡牌效果编辑区；中间为卡片预览区，可直接拖动卡图位置并用滚轮调整卡图缩放；右侧为全局卡牌库，显示可用于 `/moonspire give_card` 的卡牌 id。卡面页左侧为 id、卡面图片入口和区域选择；中间为卡面预览区，可直接拖动费用、名称、卡图、类型和描述区域；右侧为卡面列表。
- 新建或调整开发者中心界面时必须先确认三条底线：所有可见文案都必须走翻译 key，不能直接把 `debug.moonspire.*` key 露给玩家；所有列宽和面板边界必须按屏幕可视区反推，不能让列表、搜索框或内容框越过右侧安全边界；所有预览面板的外框必须跟随列本身的边界绘制，不能只按内部卡片或内容居中位置画框，避免出现底边缺失、面板漂移或内容超出屏幕。
- 已知复发错误：卡面页曾出现左侧直接显示翻译 key、预览面板少底边、右侧列表越界这三类问题。以后做任何 UI 页面都要先检查是否存在“未翻译文案”“面板边框按内容而不是按列绘制”“列宽未被屏幕边界约束”“新增状态文本没有单独预留行高”这四种风险；只要新增预览区、列表区或状态提示区，就必须先定义列边界和底部预留，再放内容，不要靠后补裁剪修正。
- 卡牌图片有三个入口：`本地图片` 在后台线程优先通过独立的 Windows PowerShell STA 进程打开系统原生 PNG 文件选择窗口，失败时才回退到置顶 owner 的系统原生窗口；选中 PNG 后先引用原路径即时预览，选择结果会校验为 PNG 文件，保存时会把卡图烘焙并复制到版本目录 `moonspire/developer/card_art/<card_id>.png`，同时把该卡的 `artPath` 切换为对应的相对文件名，后续渲染直接引用该目录里的图片；再次更换卡图时会覆盖同名 PNG，清空/重置卡图时会删除对应的生成图片，避免留下旧文件；`物品图标` 打开可搜索、可滚动且不再按旧分类过滤的物品图标弹窗，弹窗顶部必须显示并独立渲染物品搜索框，关闭后不残留搜索框；`重置` 清空开发者覆盖图并恢复默认或无卡图。
- 卡面图片使用与卡牌卡图一致的本地 PNG 选择入口；选中后先引用原路径即时预览，保存时复制到版本目录 `moonspire/developer/card_faces/<face_id>.png`，并把卡面 `imagePath` 切换为对应的相对文件名。卡面底图渲染始终把整张 PNG 缩放到卡牌卡面尺寸，不裁剪源图；因此横图、竖图和任意大小源图都会完整填满当前卡面。卡面页不再依赖手填图片路径作为主要入口。
- 卡面图片保存后必须让 `imagePath` 指向 `moonspire/developer/card_faces/` 下的相对 PNG 文件名，并刷新该目标文件的动态贴图缓存；否则覆盖同名卡面 PNG 后可能继续显示旧贴图或回退到默认卡面。本地 PNG 选择后要单独记录本次选择的源文件直到保存完成，保存时优先从该源文件复制到目标 PNG，并在复制失败时用翻译 key 显示具体失败原因，不能静默吞掉导致界面看起来已保存但文件未落盘。默认卡面底图仍由源码资源 `card_base.png` 提供，默认卡面 id 保持 `default`，默认卡面的当前排布坐标以 `card_faces.json` 的 `default` 条目为准，当前默认名称区域为 `x:28 y:11 width:72 height:13`。
- 卡面页的费用、名称、卡图、类型和描述区域在预览图上直接编辑：点击区域选中，拖动移动；滚轮同时调整宽高，Shift + 滚轮调整宽度，Ctrl + 滚轮调整高度；Tab/Shift+Tab 切换区域，方向键移动区域，Ctrl + 方向键调整宽高。区域数据仍保存为 `DeveloperCardFace.Area` 的 `x`、`y`、`width`、`height`。卡面预览图上的区域文字按各自区域水平和垂直居中绘制，用于直接预览实际排版效果；费用区域固定显示数字 `1`，便于观察费用槽对齐效果。
- 卡面页底部按钮栏除通用 `保存`、`完成` 外，还提供卡面页专用的 `新建`、`删除`、`重置`、`应用`。`删除` 必须使用 `MoonSpireModalLayer` 顶层确认弹窗，并在确认或取消关闭时清理底层 hover、拖拽和滚动条抓取状态。删除非默认卡面后，所有使用该卡面的卡牌回退到默认卡面并选回默认卡面；删除默认卡面时恢复模组内置默认卡面。
- `应用` 会打开独立全屏卡面应用选择器，而不是 modal 弹窗。该界面继承 `NoBlurScreen`，顶部是搜索框，中间是卡牌列表，底部固定显示 `选中所有`、`取消所有`、`重置`、`应用`、`完成` 五个按钮。卡牌列表必须限制在屏幕边界内，并为搜索框、滚动条、第一排/最后一排卡牌完整显示和底部按钮预留明确空间；按钮文字保持居中，长卡名在卡牌下方按列宽缩放显示。
- 卡面应用选择器中，点击卡牌切换选中状态，再次点击同一卡牌取消选中；只有这种开发者可选择场景中的已选卡牌绘制蓝色边框。该蓝色边框不代表战斗目标、手牌选中或普通卡牌 hover 状态，不能扩散到非选择器的战斗 UI。
- 卡面应用规则：点击 `应用` 后，所有选中卡牌改用当前卡面；原本使用当前卡面但这次未选中的卡牌回退到默认卡面；若卡牌原本使用其它卡面而这次被选中，则取消其它卡面对该卡的应用并改为当前卡面。默认卡面等价于未特殊应用，不应为回退到默认的卡牌保留多余的 `faceId: default` 覆盖记录。
- 卡牌渲染卡面时必须优先按当前开发者数据中的规范化卡牌 id 解析最新卡面应用，其次回退到基础卡定义的卡面，最后才回退到 `CardInstance.faceId`。这样卡组、手牌、抽牌堆和弃牌堆中已经存在的旧 `CardInstance` 不需要重新生成，也能在卡面应用保存后显示最新卡面；开发者中心和卡面应用选择器必须把当前会话中尚未保存的 `DeveloperData` 显式传给卡牌渲染，保证点击 `应用` 后预览即时变化；不要只依赖实例里保存时的 `faceId` 或全局已保存数据。
- 渲染开发者卡牌实例时，如果 `CardInstance` 同时有注册 id 和 `developerCardId`，必须优先使用 `developerCardId` 匹配当前会话里的 `DeveloperCardDefinition`；否则 `custom_`/覆盖卡牌会被基础注册 id 抢先匹配，导致卡面应用后开发者中心预览仍显示默认卡面。
- 卡牌效果区通过 `+` 打开可搜索的效果列表，弹窗必须限制在开发者中心可视区域内并避开底部按钮栏，且作为最上层内容绘制，用实心暗色面板盖住并吞掉底层输入框、列表和卡牌交互；切换卡牌、页签或打开其它弹窗时应关闭效果列表。当前提供造成伤害、获得格挡和施加流血。每条效果行左侧 `-` 会移除该效果，中间显示效果名称，右侧只在该效果需要目标且数值有效时显示可点击的 `目标` 按钮，否则显示灰色目标文本；点击 `目标` 会打开同款可搜索的目标列表选择器，列表包含自身、单体敌方、单体友方、全体敌方、全体友方、全体单位、自身以外的所有单位和随机敌方，选中后回填当前效果行。目标选择器与效果列表一样必须作为最上层内容绘制、吞掉底层输入，并在内容溢出时支持连续滚动和滚动条拖拽。数值输入框直接编辑单次效果数值，次数输入框编辑该效果重复次数，次数默认为 1。只有带数值的效果能编辑次数。删除后数值和次数输入框必须同步移除。卡牌被切换选中、创建或恢复时，这组输入框要按当前卡牌效果列表重新生成，不能只保留上一次选中卡牌留下的控件。卡牌效果区内容相对边框内缩显示，并且卡牌效果区、效果选择弹窗和目标选择弹窗都要在内容溢出时支持连续滚动和滚动条拖拽，不允许盖住底部按钮栏。效果列表会转为卡牌统一效果列表、目标数据和次数数据。
- 开发者中心使用左侧固定页签栏、中间固定表单区、右侧固定列表/预览区和底部固定按钮栏。左侧页签按钮必须按侧栏面板宽度水平居中；每个输入框上方必须绘制固定字段标签，不能只依赖占位文本说明字段含义。
- 开发者中心保存后重新接回同一界面时必须保留当前页签与对应选中状态，不得把卡面页等当前工作页强制跳回卡牌页；卡牌页保存后必须保留右侧选中的同一张卡，哪怕数据刷新后重新构建界面也一样；怪物页还必须保留右侧怪物列表的滚动位置。
- Moon Spire UI 的按钮文字默认保持水平居中和垂直居中；除非某个特殊按钮明确要求图标或文本左对齐，否则 `MoonSpireTextureButton` 和手绘按钮都必须把文字画在按钮中心，长文本需要在按钮内裁剪或缩放后仍保持居中。
- Moon Spire 纹理按钮必须在渲染时用当前鼠标坐标做实时命中判断，并在命中时绘制 `button_hovered.png`；不要只依赖可能未及时更新的旧 hover/focus 状态。新增或改动开发者界面按钮后，必须确认鼠标指向按钮时贴图会切换到高亮态。
- 开发者中心中没有注册成 widget 的手绘按钮，例如本地图片、物品图标、重置、卡面区域选择、效果加减和弹窗按钮，也必须使用当前鼠标坐标绘制高亮贴图，不能只修 `MoonSpireTextureButton`。
- 卡牌、卡面、怪物和战斗 UI 四个页签使用各自独立的布局规则：卡牌页包含中间表单、卡片预览、物品选择弹窗和卡组式小卡牌网格；卡面页包含表单、卡面预览和卡面列表；怪物页包含表单和怪物列表；战斗 UI 页只显示布局说明入口，不显示无关输入框。
- 怪物页的卡组字段不再使用可手填的卡牌 id 输入框，而是显示 `打开卡组` 按钮和当前有效卡组张数。未保存覆盖时，怪物页和怪物卡组界面读取该实体的默认战斗数据与 5 张默认卡组规则作为显示基础，外侧张数也显示这 5 张默认卡；已有覆盖只替换对应字段，卡组只在用户实际编辑后写成覆盖。点击后打开独立的怪物卡组界面：主界面只允许同时选择一张卡，底部固定显示 `添加`、`删除`、`复制`、`重置`、`完成`；`添加` 会进入全卡牌列表界面，该界面顶部有搜索框、允许同时选择多张卡，底部固定显示 `添加` 和 `取消`。确认添加后把所选卡牌追加到怪物卡组并返回主卡组界面，取消则不改动并返回主卡组界面。两个卡组界面都继承 `NoBlurScreen`，使用箱子式暗色背景、可拖动滚动条和卡牌蓝色选择边框。
- 怪物页底部提供 `重置` 和 `删除`。`重置` 将当前怪物覆盖恢复为上一次保存状态；`删除` 必须通过顶层 modal 确认，确认后移除当前怪物覆盖，使该怪物的血量、费用、速度和卡组回到默认状态。
- 开发者中心底部按钮栏中，`保存` 和 `完成` 是所有页签共用的固定按钮，其余操作按钮必须按页签单独创建。开发者中心主界面、卡面应用选择器和怪物卡组界面的底部按钮统一使用 64x20 尺寸和 8 像素间距；左侧页签按钮使用 90x20 尺寸并按侧栏面板水平居中。页签专用按钮按可用区域居中，通用 `保存`、`完成` 靠右成组显示。卡牌页底部显示 `新建`、`删除`、`重置`、`筛选` 和 `给予`；`给予` 会把当前选中卡牌加入当前玩家卡组。卡牌库展开/收起使用卡牌列表左侧的小按钮，展开显示 `《`，收起显示 `》`，不再占用底部按钮栏；该按钮使用独立 `card_list_toggle.png`，其上下左右中心固定在卡牌库面板左边界与内容区垂直中线的交界处。`重置` 会把当前选中卡恢复到本次打开开发者中心后最近一次成功保存时的状态；若该卡是保存后新建且未保存的临时卡，则移除该卡并选回可用卡牌。删除必须先显示确认弹窗，确认弹窗必须作为最顶层 modal 绘制并吞掉底层输入，底层按钮不能盖住弹窗；确认后按规范化卡牌 id 删除开发者自定义/覆盖定义；卡牌页列表和删除后的选中回退必须以当前会话 `data` 叠加内置/转换基线卡为准，不能从已保存的开发者覆盖回读旧卡；内置怪物牌、转换牌和模组本体卡删除覆盖后会保持选中同一张卡并立即回退到默认定义，自定义卡删除后会选回当前筛选下的其它可用卡。删除弹窗确认或取消关闭时必须清理底层按钮 hover、拖拽和滚动条抓取状态，避免鼠标目标取消后仍残留瞄准/高亮。
- 右侧卡牌库筛选包含全部、模组本体、怪物、转换和自定义，不再带费用筛选。缩起时使用原本右侧宽度，展开时占满除左侧表单外的剩余宽度，高度不变；卡牌绘制区域必须给右侧滚动条预留命中宽度，卡牌和滚动条不得重叠。右侧区域较窄时，小卡牌和长卡牌 id 都要按可用列宽自动缩小显示，不能画出面板或滚动条区域；最后一行需要额外滚动空间避免被底部裁掉。
- 卡牌页没有选中卡牌时，左侧编辑字段、卡牌图片操作、效果编辑区和中间预览都不展示，也不能保留上一次选中卡牌的文字或预览残留；进入卡牌页默认也应保持无选中状态。点击右侧卡牌库空白处、搜索/筛选后当前卡牌不可见时，都按无选中状态处理，只保留右侧卡牌库和筛选信息。
- 右侧卡牌库第一行也必须有明确顶部内边距，且渲染、点击命中、首尾可见行计算和滚动内容高度必须使用同一套顶部/底部 padding，避免最顶上一张卡牌被搜索栏下方裁剪。
- 开发者中心的物品网格、卡牌网格、卡面列表、怪物列表以及其它可能超出可视范围的内容区必须遵守通用滚动条规则：内容放得下时不显示滚动条，内容溢出时允许滚轮滚动和直接拖动滚动条。
- 开发者中心的列表区域底部必须为按钮栏预留空间，列表点击范围和绘制范围都不能进入底部按钮栏。长卡牌 id、怪物 id、物品名和说明文字必须在各自列宽内截断或按可用高度换行，避免覆盖输入框、说明文字或按钮。
- 开发者数据主文件位于 `moonspire/developer/ui/developer_data.json`；保存时还会按功能同步写出 `card_faces/`、`cards/`、`monsters/` 子目录文件，卡牌最终 PNG 图写入 `moonspire/developer/card_art/`，卡面最终 PNG 图写入 `moonspire/developer/card_faces/`，并在保存后把卡牌数据中的 `artPath` 和卡面数据中的 `imagePath` 切换为对应的文件名。

## 手牌卡牌与开发者效果行

- 手牌中的小卡描述必须限制在卡牌描述区域内绘制，文字不能穿透到覆盖它的其它手牌上方。手牌悬停、拖拽指向目标、怪物已打出牌和卡牌预览使用完整卡牌尺寸的详细渲染参数，描述区域按可用高度显示更多行，同时卡图走不裁剪路径，避免缩放姿态下误裁物品图。
- 怪物已打出牌必须和怪物意图牌、玩家手牌一样按当前战斗双方速度、格挡和防护状态计算预览数值；即使该牌来自视觉事件、已经不在怪物手牌或意图列表中，也要显式按怪物作为出牌者渲染伤害/格挡的增减颜色。
- 手牌选择效果打开时，`BattleScreen` 必须把该界面作为真正的顶层阻塞 modal：底层战斗条目、怪物意图、底部牌堆/费用/按钮、底层手牌、已打出的牌、出牌飞行动画、底板各区域文字、HUD 提示、牌堆提示、布局编辑器文字和基础 widgets 都不得绘制到遮罩上方，也不得响应出牌、结束回合、牌堆、目标、拖拽、右键和 ESC。代码应先跳过这些底层层级，再使用 `MoonSpireModalLayer.drawTopmostOverlay(...)` 绘制箱子式暗色阻塞层，最后只重绘 modal 自己拥有的候选手牌、选中手牌、标题和确认按钮。只有候选手牌和确认按钮可点击。
- 手牌选择层中的候选手牌按统一手牌布局规则排布：只要牌脱离手牌区域（被选中、拖拽或进入其它结算位置），剩余手牌立即按当前仍在手里的牌重新收紧；只要牌返回手牌区域，整组手牌立即重新铺开。被选中的手牌使用帧时间动画移动到确认按钮上方，多张牌并排显示；再次点击返回手牌区。候选牌悬停放大使用普通手牌悬停预览的缩放和屏幕边界钳制规则，不能超出屏幕，描述和关键词提示必须位于遮罩上方。未选够时顶部标题显示选择要求且确认按钮使用禁用纹理；选够后标题切换为确认文案，按钮切换为可用/悬停纹理。确认后选择层进入确认等待状态并播放结算动画：丢弃的牌从当前位置飞向弃牌堆，被消耗的牌留在原地播放消耗动画，当前实现为原地淡出，不得飞向消耗牌堆；等待期间仍绘制遮罩、候选/选中手牌、标题和禁用确认按钮，遮罩直到服务器快照清除本次选择流程后关闭。
- 开发者中心卡牌效果行的数值输入框前必须显示“数值”标签，次数输入框前必须显示“次数”标签；两个标签均使用翻译 key，不能写死中文或英文文本。新增标签后，目标按钮、数值框、次数框仍需在表单网格中从右向左排布，不能挤出滚动内容区域或影响弹窗命中。

## 手牌重叠与效果行间距

- 普通战斗手牌处于扇形重叠状态时，小卡仍保留效果描述正文，但描述文字单独作为后置文本层绘制，并按后续前景手牌的实际屏幕包围盒切分成可见片段。被前景手牌覆盖的描述片段不绘制，露出的片段继续显示；悬停放大、拖拽详细卡和其它详细预览始终显示完整描述。这样既保留普通手牌信息，又不会让下层文字透过半透明卡面显示在上层卡牌上。
- 开发者中心卡牌效果行必须为左侧删除按钮和右侧目标按钮保留最小安全间距。效果行从左到右依次为删除按钮、可截断的效果名、固定宽度目标按钮、紧贴输入框的数值标签、窄数值输入框、紧贴输入框的次数标签和窄次数输入框；右侧数值/次数控件锚定在效果网格可视宽度内，不进入滚动条命中区，并优先压缩输入框宽度给效果名和目标按钮留空间。效果输入框必须在效果列表自己的裁剪区域内单独渲染，滚动到半行可见时仍显示可见部分，不能因为未完整进入视口而整框消失。效果列表底部保留安全内边距，最后一行和 `+` 按钮不能贴住表单边框。
## 统一遮罩层

- 所有 Moon Spire 自绘的模态、弹窗、选择器、牌堆/卡组查看和战斗阻塞选择层，都必须使用箱子/物品栏式黑色透明遮罩。
- 遮罩统一通过 `MoonSpireUiTextures.drawOverlay(...)` 或 `MoonSpireModalLayer.drawTopmostOverlay(...)` 绘制，不允许使用灰色近乎不透明遮罩来掩盖层级问题，也不允许添加模糊效果。
- 绘制顺序必须是：底层界面正常作为背景渲染，随后绘制黑色透明遮罩，最后绘制当前模态内容。
- 遮罩打开时，底层战斗 UI 可以作为背景可见，但不得响应输入；底层文字、卡牌描述、hover 放大、已打出的牌、布局编辑文字和 tooltip 都不能绘制到遮罩之上。
- 战斗中的手牌选择层和牌堆查看层按 ESC 时打开暂停/设置菜单，不关闭当前选择或牌堆流程。
- 所有区域数字和底层 HUD 文字都属于遮罩下方内容，包括费用数字、抽牌/弃牌/消耗牌堆数量、角色生命/速度/意图数值、世界头顶伤害/效果数字、卡组/牌堆网格标题和数量、开发者布局文字等；这些内容不得使用高 z、`SEE_THROUGH` 字体模式或 see-through 渲染类型穿过遮罩。
- 费用数字、抽牌/弃牌徽章数字、消耗牌堆数字和结束回合按钮文字必须作为普通 GUI 文本绘制在底层；`MoonSpireUiTextures.drawOverlay(...)` 会先提交已经排队的底层 GUI 字体内容，再在高于底层 GUI 的 z 层绘制箱子式半透明遮罩并立刻提交遮罩本身，确保这些数字被遮罩压暗而不是穿透到遮罩上方。
- `MoonSpireUiTextures.drawOverlay(...)` 是唯一的整屏遮罩入口，绘制时必须压住之前已经提交的低层 GUI/世界文字，且遮罩提交完成后才能恢复后续 modal 内容绘制；如果某个数字或提示穿过遮罩，必须降低该元素层级、修正提交顺序或修正遮罩 z 层，不能通过提高遮罩不透明度解决。
- 模态打开期间，底层 hover 预览、放大卡牌、关键词提示、已打出的牌、飞牌动画和 tooltip 默认不绘制；只有当前模态拥有的卡牌、标题、按钮、预览和提示可以在遮罩上方绘制。
- 牌堆/卡组/弃牌/消耗牌网格也遵守同一规则：网格自己的标题、计数、卡牌和放大预览属于该网格模态内容；其背后的战斗 UI、其它网格或开发者底层文本不得越过遮罩显示。
- 战斗模态打开时，基础战斗条目整体不渲染；不要只隐藏血条文字或速度数字，因为基础条目的血量、速度、意图和效果数字都属于遮罩下方内容，必须避免重新穿层。
- 战斗与卡组相关界面不得在普通开发者模式下持续进行逐帧性能统计或日志采样；需要排查卡顿时只能临时开启明确的诊断入口，完成后应移除或关闭，避免诊断本身增加延迟。

## 手牌选择层

- 手牌选择层只允许候选手牌和确认按钮点击，禁止出牌、拖拽、结束回合、目标选择、牌堆入口、右键和底层 widgets。
- 未选够时标题显示选择要求，确认按钮禁用；选够后标题切换到确认文案，确认按钮启用。
- 选中手牌会从手牌区平滑移动到确认按钮上方，多张并排显示；再次点击返回手牌区。
- 手牌脱离或回到手牌区时，剩余手牌立即按统一手牌布局规则收紧或铺开。
- 确认后选择层进入确认等待状态并播放结算动画：丢弃牌飞向弃牌堆，消耗牌在当前位置淡出，不飞向消耗堆；等待期间继续绘制遮罩、选中手牌和禁用确认按钮，服务器快照清除选择流程后关闭遮罩。手牌选择层自己的卡牌、标题、按钮和预览必须在共享遮罩绘制完成后整体提升到 modal 内容层绘制，避免被遮罩的高 z 层或深度状态压住；底层战斗 UI 仍作为背景保留在遮罩下方，不得通过隐藏底层 UI 修复穿层。

## 牌堆与卡组预览

- 弃牌堆、消耗堆、卡组等网格界面的放大卡牌必须保留顶部和底部安全区；放大位置以原网格小卡中心为锚点，不得因为放大后超出屏幕边缘而向屏幕中间回弹，也不得被确认按钮遮挡。放大卡牌的关键词提示框仍需显示在屏幕内可见位置：优先贴近放大卡牌左右侧，若卡牌自身越界则只钳制提示框坐标，不反向移动放大卡牌。
- 卡组/牌堆网格的滚动条位置保持贴近原右侧位置；卡牌本体按一行可容纳 7 张的空间缩小计算，但实际网格仍只排 5 列。放大预览必须随网格卡牌同比例缩小，而不是继续使用完整 128x158 大卡尺寸。
- 牌堆/卡组放大预览属于当前网格模态自己的顶层内容，可以盖过该模态内的普通按钮；顶部标题和数量区域会作为同一模态的顶层遮罩最后重绘，因此放大卡牌和关键词提示进入顶部标题区时必须被该顶部遮罩压住。关闭预览时必须在动画低于可见阈值后立即清除整张预览，避免费用数字或卡面局部残留。
- 卡牌网格应缓存卡牌列表、布局结果和当前可见索引范围；打开卡组/牌堆时不使用会隐藏真实卡牌内容的占位卡面，必须保持完整卡牌渲染、悬停放大和关键词提示可用，同时通过缓存战斗卡组列表和网格布局减少大列表打开与滚动时的卡顿。卡组/牌堆网格自己的卡牌、滚动条、标题和放大预览必须在共享遮罩绘制完成后整体提升到 modal 内容层绘制，遮罩下方的战斗 UI 背景保持可见但不得穿到网格内容上方。开发者中心、卡面应用选择器和怪物卡组列表应缓存筛选结果，减少打开大列表时的卡顿。
- K 键打开的玩家卡组界面复用 `CardGridPanel` 的箱子式暗色遮罩、标题计数、滚动条和放大预览规则。非战斗 `DeckScreen` 只在没有其它屏幕时打开，并可再次按 K 关闭；战斗中只允许在基础 `BattleScreen` 且没有其它 modal/编辑器时打开自己的当前战斗卡组，打开后再次按 K 关闭该卡组界面。战斗中由 K 打开的卡组网格每帧从最新战斗快照刷新手牌、抽牌堆和弃牌堆内容；抽牌堆、弃牌堆、消耗堆和右键查看手牌等其它网格弹窗不响应 K 关闭。
## 本次战斗 UI 布局补充

- `player_entry` 和 `monster_entry` 现在表示左右两侧的条目区起点与条目宽度。条目本身使用固定 42 像素高度，条目区宽度跟随该基础条目的宽度；条目区底部默认延伸到消耗堆上方 8 像素处。
- 条目数量超过可见高度时，鼠标位于对应条目区内可滚轮滚动。该滚动不显示滚动条，点击、右键查看和卡牌指向命中都会使用同一套滚动偏移与裁剪区域，避免看得到但点不到或点到隐藏条目。
- 条目区只显示存活战斗单位。假死亡单位的条目会从列表中移除，世界中的选中轮廓、头顶血条、意图、状态、格挡动画与伤害数字也会停止显示；拖卡与点击选中不会再命中假死亡单位。
- 指向/拖卡高亮优先于已选中高亮。拖卡指向某个目标时不会再同时保留旧选中怪物的黄色高亮，避免“当前指向目标”和第一只怪物一起高亮。
- 牌堆查看和手牌选择等 modal 关闭前后，底层战斗 UI 会作为背景继续绘制在标准箱子式黑色半透明遮罩下，避免退出丢牌/消耗牌界面时出现短暂空屏。
- 多怪物意图 UI 会根据当前指向或选中的怪物切换数据来源，意图小卡、放大预览和条目旁意图汇总都读取对应怪物自己的意图列表。
