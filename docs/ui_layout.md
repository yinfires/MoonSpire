# Moon Spire UI 布局与 PNG 资源说明

Moon Spire 的自定义界面底图都放在：

`src/main/resources/assets/moonspire/textures/gui/ui/`

战斗效果图标等状态类 PNG 单独放在：

`src/main/resources/assets/moonspire/textures/gui/effects/`

战斗过程中的一次性动画 PNG 单独放在：

`src/main/resources/assets/moonspire/textures/gui/animations/`

这些 PNG 是当前仍有有效代码路径的模组重绘材质入口。未被 UI 代码引用的旧贴图会从资源目录移除；后续需要恢复某个独立视觉样式时，应先重新接入渲染代码，再加入对应 PNG。卡牌本体底图仍在：

`src/main/resources/assets/moonspire/textures/gui/cards/card_base.png`

## UI 渲染性能规范

- 所有 Moon Spire UI 热路径默认优先剔除玩家完全不可见的内容：整张卡、整行条目、滚动窗口外内容、完全离屏区域、完全被 modal 遮罩或前景 UI 拥有的底层内容都应跳过渲染。可见性判断应先使用低成本包围盒、列表可见范围或 modal 所有权，不应在热路径中为了少量重叠像素制造大量状态切换。
- 当自然绘制顺序已经能正确遮挡内容时，优先依赖绘制顺序，不要预先把底层卡图或文字切成多个片段。scissor/分片绘制只适合滚动窗口、明确裁剪边界或少量大区域裁剪；普通手牌重叠、HUD 叠放和高频文字渲染不得因为局部遮挡反复启用 scissor、立即 flush 或重复提交字体。
- 玩家能看到的内容必须保持真实完整，不得用占位图、实色遮挡、离屏整卡缓存、跳过可见文字或跳过真实物品卡图换取帧率。滚动中的卡组/牌堆网格可以在少量滚动运动帧内依赖外层滚动视口裁剪物品卡图，暂时省掉逐卡物品卡图 scissor；这只允许用于居中缩放的 `renderFakeItem` 物品卡图，不得跳过真实卡图、卡框或文字，也不得影响自定义 PNG 卡图的逐卡裁剪。任何“只渲染可见内容”的优化都必须同时满足三点：渲染不缺块、性能日志证明更快、每帧 GUI 提交和字体提交次数没有上升。
- 普通战斗手牌必须保持完整文字规范：每张可见手牌都按同一规则渲染费用、名称、类型和可见描述；若描述文字经可见性判断确实完全离屏，可以跳过描述提交，但不能仅因手牌处于后排、未悬停或重叠状态而隐藏类型或描述。

## 开发者模式

创建或编辑：

`config/moonspire/client.json`

内容示例：

```json
{
  "developerMode": true
}
```

`key.moonspire.ui_debug` 默认绑定为 F8，但只有客户端启动时 `config/moonspire/client.json` 已开启 `developerMode` 才会注册到 Moon Spire 按键分类中；关闭开发者模式时，Controls 中不会显示开发者中心/调试入口，也不会留下问号占位项。按下该键时仍会重新读取 `developerMode` 作为二次保护；若此时开发者模式已关闭，F8 会静默无反应，不显示提示、不打开界面、不向服务器发包。若从关闭改为开启，需要重启客户端后才会在按键控制中出现 F8 入口。编辑器仍然只能在战斗进行中打开；普通玩家未开启开发者模式时不会看到布局编辑器。

战斗界面只会在没有其它 Screen 打开时由客户端自动补回。暂停菜单、退出世界、断开连接等原版界面流程不应被战斗界面覆盖；客户端退出或世界为空时必须清空本地战斗状态和布局编辑器状态。

战斗界面本体不绘制暗色透明背景，必须保持世界画面直接可见。只有卡组查看、牌堆弹窗、布局编辑器等覆盖层可以使用 `panel_dark.png` 或其它暗色透明底；整屏遮罩由代码直接绘制，不再依赖独立 `overlay.png`。

## 布局字段

每个布局元素包含：

- `anchor`：锚点，可用 `top_left`、`top_right`、`top_center`、`bottom_left`、`bottom_right`、`bottom_center`、`center`、`mouse`。
- `x`、`y`：相对锚点的偏移。
- `width`、`height`：元素基础尺寸。
- `scale`：最终缩放倍率。

稳定战斗布局元素 id：

`player_entry`、`monster_entry`、`monster_intent`、`energy`、`draw_pile`、`discard_pile`、`exhaust_pile`、`end_turn`、`hand`、`play_area`、`card_preview`、`tooltip`。

旧版调试元素 `pile_overlay`、`world_bar` 和 `world_effects` 已移出布局 schema。加载旧覆盖文件时，代码会忽略这些无效 id，避免它们重新出现在布局编辑器中；世界头顶血条和效果仍由 `BattleWorldOverlay` 在实体 billboard 坐标内固定绘制，不属于屏幕 UI 布局编辑范围。

## 编辑器操作

- 点击 UI 元素选中；当前选中 UI 的黄色轮廓会最后绘制，避免被其它 UI 轮廓覆盖。
- Tab 选择下一个 UI 元素。
- Shift + Tab 选择上一个 UI 元素。
- 拖动选中元素调整位置。
- 滚轮调整缩放。
- Shift + 滚轮调整宽度。
- Ctrl + 滚轮调整高度。
- 方向键移动 1 像素。
- Shift + 方向键移动 10 像素。
- Ctrl + 方向键调整宽度或高度。
- Ctrl + S 保存覆盖文件。
- Ctrl + R 重新加载布局数据。
- Ctrl + D 重置为默认资源布局。

## 覆盖层输入与背景

- 战斗内的抽牌堆、弃牌堆和卡组查看覆盖层打开时，覆盖层本身负责吞掉鼠标点击、拖拽、滚轮和键盘输入，并清空底层战斗 hover、拖牌、相机拖动和待选目标状态；右键或 ESC 只关闭当前覆盖层。抽牌堆和弃牌堆的 HUD 提示框必须避开被指向的牌堆图标矩形，不能遮挡玩家正在查看或点击的牌堆入口。
- 卡牌网格覆盖层的暗色背景由 `MoonSpireUiTextures.drawOverlay()` 直接绘制为箱子式暗色透明遮罩，不依赖 PNG 像素透明度，因此不能呈现为完全不透明的纯黑背景。该 helper 必须在绘制前提交底层 GUI，再绘制遮罩并立刻提交遮罩本身，随后才允许 modal 标题、按钮、卡牌和提示绘制到遮罩上方。
- 胜利后的卡牌奖励界面是独立 `NoBlurScreen`，由 `MoonSpireUiTextures.drawOverlay()` 绘制暗色透明背景，不使用模糊。顶部只绘制 `选一张牌` 标题文字，不绘制卷轴、面板或其它标题背景，标题和底部 `跳过` 按钮都向屏幕中部内收，避免贴近上下边缘；中间默认显示服务端抽出的 3 张奖励牌，少于 3 张时只显示实际数量，普通奖励牌尺寸为完整卡牌的约 2/3。卡牌悬停时在原卡牌附近放大并显示关键词详情，点击卡牌或点击跳过都会关闭当前奖励页并进入下一只战斗开始敌方的奖励页。
- 战斗内牌堆覆盖层不再使用可调布局元素，而是由 `CardGridPanel` 按当前屏幕宽度直接铺开；抽牌堆、弃牌堆、消耗堆和右键查看实体战斗牌库等战斗内牌堆与 `DeckScreen` 一样不再使用底部预留，不在底部额外绘制遮罩或压缩可视高度。
- `play_area` 控制非指向目标卡牌的有效出牌区域。玩家拖动格挡、自身收益、全体或随机目标等不需要箭头精确命中的卡牌时，鼠标进入该矩形才会显示可打出反馈并允许释放；区域底部仍会被手牌区域顶部略微截断，避免在手牌上误触发出牌。
- 怪物意图小卡和放大预览卡牌本体会阻止底层怪物目标选择与 hover 更新，避免查看意图牌时选中怪物或造成预览闪烁；关键词提示区不参与 hover 保持判定。
- 中央 `monster_intent` 意图区卡牌默认不显示；只有鼠标 hover 或当前选中单位拥有意图时，才显示该单位的意图卡。玩家方非本地友方意图可在玩家回合被读取，玩家方条目旁意图汇总绘制在条目右侧；敌方条目旁意图汇总继续绘制在条目左侧。
- 战斗中的 hover/拖牌高亮会同时作用于条目和世界实体。玩家不会被点击选中，但会在鼠标指向或自我收益牌进入有效出牌区域时显示蓝色轮廓；怪物仍可被点击选中并显示黄色轮廓。若卡牌效果里包含单体敌方或单体友方，就必须拖动箭头实际指向对应目标，并且拖拽高亮只保留当前指向的单体目标，不能同时保留已选中的第一只怪物。全体友方、自身以外的所有单位等范围目标不要求拖箭头。单体友方不包含自己，当前 1v1 战斗里没有其它友方时不会把玩家当作合法单体友方目标。如果同一张牌有多条效果，则进入有效出牌区域时按实际会高亮的目标数量选择“覆盖单位数量最高”的目标组，当前以单体、自身、全体敌方、全体友方、全体单位、自身以外的所有单位和随机敌方目标为准。条目区滚动裁剪必须给蓝色 hover 框和黄色选中框保留边缘像素，不得切掉第一行或最后一行的外框。
- 需要拖拽箭头指向单体目标的手牌在瞄准期间必须固定显示在手牌区的屏幕中心线上；该位置只能继承当前手牌布局的缩放和底部高度，不能用“中间那张手牌槽位”的 x 坐标覆盖中心点，否则偶数手牌、扇形展开或布局偏移时会出现左右偏移。
- 怪物正在使用的牌绘制在 `monster_intent` 布局区域中心，尺寸只比意图小卡略大，且会在屏幕内自动夹紧，避免战斗窗口较小时卡面越界；该牌仍然是最顶层表现层，不会被其它卡牌遮住。

## 滚动条规则

- 所有带滚动条的 Moon Spire UI 都只在内容无法完整显示时绘制并允许滚动条滚动。卡牌奖励界面未来若显示超过 3 张候选牌，滚动条必须出现在最右侧卡牌外侧，不能覆盖卡牌本体或悬停预览。
- 所有可能出现多内容且显示不全的 Moon Spire UI 区域都必须支持滚动；内容溢出时至少要支持鼠标滚轮滚动，并在使用滚动条呈现时支持滚动条拖动。
- 所有多内容区域都必须为内容的上、下、左、右完整可见预留内边距，并把该内边距纳入内容高度、裁剪区、滚动上限和点击命中计算；不能让第一行或最后一行卡牌、图标、文字、选中边框被裁剪区切掉。
- 滚动必须是连续像素滚动，不能按页或按整行跳转。
- 滚动条滑块必须支持直接拖拽；拖拽位置到滚动偏移的映射必须保留小数精度，不能把滑块 y 坐标先取整后再换算滚动值。鼠标继续拖向顶部或底部但内容滚动偏移已经贴边不再变化时，不能继续把这些帧当作滚动运动帧，也不能持续刷新滚动诊断事件。
- 滚动后不能跳过某一行或让某一行永久看不见。
- 内容能完整显示时不显示滚动条，滚轮不会移动内容；内容不能完整显示时，滚动位置直接限制在顶部到底部的合法范围内，不做越界或回弹。
- `CardGridPanel` 负责卡组、抽牌堆、弃牌堆等卡牌网格滚动；接收卡牌内容后默认按客户端当前显示名称排序，只改变查看顺序，不改变服务端抽牌、弃牌、消耗或结算顺序。网格优先按 5 列显示，并会缩小网格卡牌以保证常规卡组宽度下一行能容纳 5 张。卡牌悬停放大以原卡牌中心为锚点绘制，并跳过原位小卡，避免叠加造成额外发光；离开悬停后会平滑缩回原位，不会瞬间消失。已有放大预览时，只按当前实际放大的卡牌本体保持预览，原位小卡只用于首次进入或切换到另一张卡；卡牌主体命中只包含真实卡面矩形，不包含可打出高亮、选中边框、脉冲外框或任何外扩滞后区。鼠标离开放大卡牌本体后，如果命中其它卡牌真实卡面，应立即切换到该卡牌；如果仍只停在原小卡或高亮边框上，则取消当前预览，避免防抖区域或高亮边框导致移出很长距离才能选中下一张牌。网格裁剪区从牌组标题和数量下方开始，只保留标题区本身作为顶部遮挡；底部不再为了按钮、边缘安全距或牌堆类型额外压缩可视高度，玩家卡组、抽牌堆、弃牌堆、消耗堆和右键查看实体战斗牌库都使用同一套无底部遮罩的可视区域，普通卡牌可绘制到屏幕底部。滚动内容顶部和底部会加入只作为空位的额外滚动距离，不作为遮盖层，使第一行和最后一行卡牌滚动后悬停放大也能完整显示。滚动时已放大的卡牌会直接贴住当前原卡牌中心移动，不产生位置追赶延迟。放大卡牌本体不再被牌堆/卡组可用边界夹动，也不会被强行移到屏幕中间。关键词提示会优先显示在卡牌右侧，右侧空间不足时显示在左侧；提示整体尽量相对卡牌高度居中，只有超过卡牌顶/底或屏幕边缘时才向下/上修正，不能反向挤动放大卡牌本体。网格打开首帧必须直接绘制真实卡牌，并复用列表建立时保存的 `contentKey`、可见范围布局和描述缓存；不得为了预热隐藏真实卡牌、显示占位卡或延迟卡面/卡图出现。
- `CardForgeScreen` 的可转换物品列表也使用相同的滚动条规则。
- `CardForgeScreen` 的可转换物品条目在鼠标指向时会在 `forge_slot.png` 槽位上方绘制亮色底和金色边框作为高亮反馈。指向某个可转换条目时，界面右侧会用完整卡牌尺寸固定绘制该物品即将转换成的卡牌详情，并沿用 `CardRenderHelper.renderKeywordTipsBeside(...)` 在卡牌左右侧显示关键词说明；这只是预览，不改变点击条目后消耗物品并生成卡牌的规则。右侧预览不新增 PNG 资源，继续复用卡牌底图、物品图标和关键词提示框；预览即时切换，不使用缩放或缩回动画，最终位置仍由制卡台布局决定。
- JEI 制卡台转换分类不新增 PNG 资源；分类图标复用制卡台物品图标，输入槽使用 JEI 标准槽背景，输出区域复用 `CardRenderHelper` 绘制转换卡牌。鼠标悬停命中只计算原输出小卡的真实卡面矩形，不把放大卡或关键词提示框纳入继续触发范围；触发展示和缩回动画可见时跳过原输出小卡，避免大小卡叠画。放大卡锚定在分类内容区内的输出列左侧空白位置，并绘制关键词提示，预览内容与 `CardForgeScreen` 使用同一套 `CardFactory.fromItem(...)` 结果；缩放过程使用 `CardPreviewAnimation`，首帧保留原输出小卡，动画卡只在可见后绘制，配方内容切换导致动画 key 不匹配时立即清空旧动画，目标位置仍由 JEI 分类自身坐标决定。

## 战斗卡牌预览

- 战斗手牌按底部浅弧形扇排显示，卡牌之间保持明显重叠；中间卡牌只略高更正，两侧卡牌轻微下沉和旋转，少量手牌必须保持平缓，不能形成尖弧或明显高拱。
- 手牌悬停预览由 `BattleScreen` 的 `HandCardAnimation` 缓存驱动，从原手牌位置平滑上抬、回正并克制放大，原小卡不再同时绘制；已有放大预览时，保持或取消必须按当前帧实际放大后的卡牌本体判断，原始手牌布局卡位只用于首次进入或切换到另一张手牌，不能用于续住当前放大牌。若鼠标离开放大卡牌本体并命中相邻手牌真实卡面，则必须立刻切换；若只停在原小卡、高亮边框、选中边框、脉冲外框或外扩滞后区上，则取消当前预览。手牌选择层不得每帧清空当前 hover 牌，否则弃牌/消耗界面和正常手牌都会在交界处反复交替放大；弃牌/消耗界面中已放大的选牌也按放大后的卡牌本体保持，不按原小卡或进入放大前的位置续住。动画差分由 `ClientBattleState` 的快照版本触发，只在战斗快照变化时同步；稳定渲染帧按真实帧间隔推进手牌与飞牌插值，并读取帧内缓存的手牌布局、hover 和目标命中，避免卡牌动画被 20 TPS 游戏 tick 限制，也避免重复布局和目标重写造成卡顿。`BattleScreen` 每个渲染帧创建一次 `CardRenderHelper.CardRenderContext`，手牌、飞牌、怪物意图、牌堆预览和抽牌飞入动画共享同一帧的开发者数据与卡面查找结果；不得在热渲染路径为了缓存命中反复构造重型卡牌签名、遍历效果列表或持续输出逐帧性能日志。
- 拖拽手牌时，从原小卡或放大预览卡牌本体按下都可以开始拖拽。拖拽开始时会立即清空当前手牌 hover 进度，拖拽期间鼠标指向其它手牌不再触发新的手牌悬停放大，也不会保留上一张牌的放大残影。技能牌和自我目标牌会以手牌小卡尺寸跟随鼠标，不再使用悬停放大的详细卡牌表现；攻击牌始终保留从卡牌指向鼠标的瞄准箭头，不会在命中怪物时把整张牌移动到目标位置。需要目标的攻击牌可以拖到怪物基础条目上直接打出，也可以在世界中直接指向怪物打出；拖拽到可打出位置时，卡牌会显示可打出背景高亮；同一次拖拽中只要卡牌离开可打出区域后再次进入，就会再次触发更强的短促高亮。若释放时没有成功打出，客户端必须把释放位置作为该手牌动画起点，让它按正常手牌插值平滑收回原位，不得直接重置到目标手牌位置。
- `BattleScreen` 通过手牌 UUID、牌堆快照和 phase 对比快照，推断抽牌飞入、非出牌手牌离场飞向弃牌堆，以及玩家/敌方回合切换提示。本地玩家打出的牌显示在战场中央；怪物、其他玩家和友方单位的出牌视觉事件统一显示在 `monster_intent` 的怪物出牌位置。服务端效果结算期间展示牌保持在屏幕中；结算结束并进入目标牌堆后，普通牌再缩小飞向弃牌堆，带“消耗”的牌和 `自爆` 查看牌必须停在展示位原地淡出，不能飞向弃牌堆。打出后的卡牌不再按当前费用显示费用不足，因为它已经不是手牌。这些都是客户端表现层，不改变服务端结算协议。
- 自爆世界动画使用客户端战斗视觉状态驱动实体缩放，并通过实体渲染层叠加白色闪烁，模拟原版苦力怕预爆白闪；该白闪不得复用受击 `hurtTime` 红色遮罩。
- 可打出的手牌会显示柔和青色高亮；费用不足或不可打出的手牌不额外变暗，只是不显示可打出高亮；卡牌费用只显示单层描边数字，字号以正常手牌小卡为基准，并随放大预览、卡组和牌堆中的卡牌实际显示比例同步缩放；费用不足时卡牌费用数字显示为红色，放大预览也必须同步变红。战斗当前费用使用费用图标加当前/最大费用比值显示，0 费使用灰色图标，有费用时使用金色图标。结束回合按钮文字使用单层描边、不带阴影；按钮内只显示四类文案：本地玩家可结束回合时显示“结束第%s回合”，敌方回合时显示“敌方回合”，本地玩家假死亡时显示“你已死亡。”，多人战斗中本地玩家已结束回合且仍需等待队友时显示“等待其他玩家结束回合”。顶部回合横幅只显示“玩家回合”和“敌方回合”，不显示“回合结束”；“玩家回合”等阶段提示不放进结束回合按钮。可结束且鼠标悬停时为红字，没有可出牌时按钮保持蓝色高亮并显示黄色文字，但按钮底图仍要根据鼠标悬停状态切换；长文案都必须在按钮内部自动缩小并保持水平、垂直居中。
- 战斗 HUD 的速度、费用、结束回合、抽牌堆和弃牌堆都会在鼠标指向时显示 `tooltip.png` 九宫格提示框，并在最顶层绘制。速度提示只绑定条目中实际绘制的速度文字矩形，矩形由当前 `player_entry` / `monster_entry` 布局位置和速度文字宽度实时计算，不能使用固定屏幕坐标；费用、结束回合、抽牌堆和弃牌堆提示分别使用对应布局元素命中范围。结束回合按钮文字追加快捷键括号，并在按钮宽度不足时等比缩小但保持居中。
- 怪物意图牌悬停预览以原小卡中心为基准原地放大，并使用与卡组/牌堆网格一致的 `CardPreviewAnimation` 平滑放大/缩回表现；只有碰到屏幕边缘时才轻微推回屏内。预览本体可保持悬停，旁边关键词提示不会扩大 hover 区域；关键词提示在左右侧选边后尽量相对卡牌高度居中，只有越出屏幕顶/底时才做最小上下修正。拖拽需要目标的攻击牌并指向怪物时，不弹出怪物意图牌预览，避免压住出牌瞄准反馈。缩回动画可见期间继续跳过对应原位意图小卡，缩回完成或当前意图列表不再包含动画 key 时立即清空动画，避免原位残留。
- 战斗选中只表示怪物意图查看：只有怪物条目和世界怪物能被点击选中并显示黄色轮廓，玩家不能被选中。鼠标 hover、拖牌目标和自我目标牌指示使用蓝色轮廓，可作用于玩家或怪物，并且条目和世界高亮必须同步。攻击牌不再使用已选中怪物作为释放回退，必须拖动箭头实际指向怪物条目或世界怪物才能打出；格挡等自我目标牌进入有效出牌区域时高亮玩家。
- 战斗外准星指向 10 格内可挑战生物时，`BattleHud` 会在准星旁绘制“按[挑战目标键]挑战目标”提示；按键名称读取当前按键绑定。该提示不使用新的 PNG 资源，只作为 HUD 文本绘制；战斗界面激活时不显示，并且按 F1 隐藏原版 HUD 时同步隐藏。提示和挑战键都使用 `ClientEvents.challengeTarget()` 的 10 格射线检测，不能依赖原版较短交互距离的 `minecraft.hitResult`。
- 非本地单位正在使用的牌绘制在 `monster_intent` 布局区域中心，使用比意图小卡略大的卡牌尺寸，并在基础战斗 UI、怪物意图小卡、手牌、飞牌和拖拽牌之后绘制，保证它是战斗卡牌层的最顶层。新牌出现时立即替换旧展示，最后一张普通牌结算完成后保留约 1 秒再移除；`自爆` 和带“消耗”的牌按消耗规则在原地淡出。该计时按渲染帧真实时间推进，且卡面数值按实际出牌者而不是固定首个怪物计算。
- 尖牙直线、尖牙环阵和召唤恼鬼的世界施法视觉由 `BattleVisualEvent` 与 `ClientBattleState` 驱动；唤魔者实体使用原版 `SpellcasterIllager` 施法状态，玩家或其它 Humanoid 使用者通过客户端临时手臂姿态表现挥手施法，不改变服务端结算。通用召唤效果共享同一套结算入口，但只有实体 id 为 `minecraft:vex` 的召唤牌继续使用唤魔者召唤施法视觉；其它召唤生物默认只按召唤物生成和快照同步显示。
- 卡牌描述中的关键词统一显示为与侧边关键词详情标题一致的较深金黄色，包括格挡、流血和守护等效果名称；卡牌描述正文使用与关键词详细描述相同的浅白色。卡牌描述会随目标变化：单体敌方和自身保留简洁写法，其它目标会明确写出“对全体敌方”“给予随机敌方”等前缀。描述内容超过卡面描述区域时，渲染层会先按当前描述框重新换行并逐步缩小字号，尽量让全部内容居中显示在区域内；只有达到最小可读字号后仍放不下时，才使用裁剪兜底。手牌、拖拽卡、怪物意图牌、卡组/抽牌堆/弃牌堆中的放大预览都会在卡牌左侧或右侧展示对应关键词详情，并按屏幕剩余空间选择展示侧。关键词和效果说明提示框必须作为最高层说明绘制，不被放大卡牌、开发者界面控件或其它普通 UI 内容压住。卡牌数值高于原值时的绿色提示使用更深的绿色，避免过亮。
- 基础条目和世界头顶条的生命/格挡使用同一套 `CardRenderHelper.combatantBarSegments()` 分段规则：先绘制底槽，再用 `health_fill.png` 绘制当前生命，用 `block_fill.png` 从生命段右侧接续绘制格挡段，生命和格挡按当前生命 + 当前格挡的有效耐久比例并排分布，避免两层填充互相覆盖或错位。世界头顶条复用 `health_bar.png` 作为底槽，底槽、生命填充和格挡填充使用固定内框尺寸；内框按 `health_bar.png` 暗槽使用 2 像素边距，填充贴图使用完整 UV，使可见色块贴合底槽暗色内框，并用轻微 Z 分层避免同平面叠加闪烁。世界头顶 UI 整体锚定在实体高度上方并额外上移，扑袭、近战冲刺和击退期间跟随实体真实位置或同一实体渲染矩阵移动，确保血条、高亮和判定位置一致；血条文字、意图数字、状态层数和伤害/治疗飘字使用普通文字渲染并位于对应贴图上方，拉近相机时也不能被状态图标或条目贴图遮挡。基础条目只保留名称、速度、血条、效果和轮廓；鼠标瞄准/悬停为蓝色边框，只有选中目标为黄色边框。
- 多人多怪战斗中，`player_entry` 和 `monster_entry` 布局元素作为双方条目的起始位置；同阵营多个条目按条目高度加 8 像素间距向下堆叠。世界头顶血条、高亮轮廓和假死亡动画会遍历所有参战单位，而不是只处理本地玩家和第一只怪物。

## 开发者中心 UI

- `key.moonspire.ui_debug` 默认仍为 F8，但只有 `config/moonspire/client.json` 开启 `developerMode` 时才注册并显示在按键控制中；它现在优先作为开发者中心入口，进入时还需要世界内拥有 2 级管理员权限。
- 战斗进行中按 F8 仍会切换战斗 UI 布局编辑器，编辑器覆盖文件保存到版本目录 `moonspire/developer/ui/battle_layout_override.json`；旧的 `config/moonspire/battle_layout_override.json` 仍会作为兼容读取来源，重置布局时新旧两个覆盖文件都会被删除。
- 开发者中心的卡牌、卡面和怪物页搜索框分别保存当前输入；保存数据后服务器回传并重建开发者中心时，当前页搜索内容和对应列表滚动位置会继续沿用。
- 开发者中心的删除确认、效果选择等 modal/弹窗必须使用统一的顶层覆盖方法：先绘制底层内容；若 modal 会覆盖底层 widget 文字，例如删除确认，则不要继续绘制底层 `renderWidgets(...)`，避免页签、按钮和输入框文字在弹窗背后保持高亮；随后绘制 `MoonSpireModalLayer.drawTopmostOverlay(...)` 的箱子式半透明暗遮罩，最后绘制 modal 本体。不要用近乎不透明的整屏黑层来掩盖底层文字问题；正确做法是让被 modal 阻断的底层 widget 不参与绘制。modal 关闭必须通过 `MoonSpireModalLayer.close(...)` 或同等的“清理交互状态后重建控件”流程，避免取消目标后底层按钮、卡牌、滚动条或拖拽状态残留高亮。
- 开发者中心包含卡牌、卡面、生物和战斗 UI 页面。卡牌页左侧为卡牌名称、指令 id、费用、卡牌图片和卡牌效果编辑区；中间为卡片预览区，可直接拖动卡图位置并用滚轮调整卡图缩放；右侧为全局卡牌库，只显示卡牌缩略图，不在网格中额外显示卡牌 id。卡牌效果编辑区的效果选择弹窗包含 `召唤生物`；召唤效果行使用实体 id 输入框代替目标按钮，允许填写如 `minecraft:vex` 或 `minecraft:silverfish` 的生物 id，数值输入框表示召唤数量，次数输入框表示召唤物存活回合数。卡面页左侧为 id、卡面图片入口和区域选择；中间为卡面预览区，可直接拖动费用、名称、卡图、类型和描述区域；右侧为卡面列表。生物页会列出可创建为 `LivingEntity` 的非玩家生物，敌对怪物默认显示内置卡组，中立和友善生物默认显示空卡组；只有添加并保存非空卡组后才会成为可挑战目标，显式清空卡组会保存为空卡组覆盖。生物页左侧还提供初始状态编辑区，使用与卡牌效果编辑器一致的加号、减号、搜索弹窗、滚轮滚动和直接拖动滚动条交互；每行显示战斗状态翻译名和层数输入框，保存到 `DeveloperMonsterDefinition.initialEffects`，战斗开始时由服务端直接应用到该生物的 `CombatantState`。
- 卡牌页的指令 id 行右侧提供复制按钮，可将用于 `/moonspire give_card` 的卡牌 id 写入系统剪贴板。卡牌库展开后会占用右侧剩余区域，但不会压缩左侧表单或改变左侧按钮范围；网格仍优先保持常规小卡尺寸，在 4 倍 GUI 缩放等窄屏显示下只压缩右侧卡牌单元和卡牌间距，确保至少一行可显示 3 张牌，并为选中框在网格裁剪区左右两侧保留可见边距。
- 新建或调整开发者中心界面时必须先确认三条底线：所有可见文案都必须走翻译 key，不能直接把 `debug.moonspire.*` key 露给玩家；所有列宽和面板边界必须按屏幕可视区反推，不能让列表、搜索框或内容框越过右侧安全边界；所有预览面板的外框必须跟随列本身的边界绘制，不能只按内部卡片或内容居中位置画框，避免出现底边缺失、面板漂移或内容超出屏幕。
- 已知复发错误：卡面页曾出现左侧直接显示翻译 key、预览面板少底边、右侧列表越界这三类问题。以后做任何 UI 页面都要先检查是否存在“未翻译文案”“面板边框按内容而不是按列绘制”“列宽未被屏幕边界约束”“新增状态文本没有单独预留行高”这四种风险；只要新增预览区、列表区或状态提示区，就必须先定义列边界和底部预留，再放内容，不要靠后补裁剪修正。
- 卡牌图片有三个入口：`本地图片` 在后台线程优先通过独立的 Windows PowerShell STA 进程打开系统原生 PNG 文件选择窗口，失败时才回退到置顶 owner 的系统原生窗口；选中 PNG 后先引用原路径即时预览，选择结果会校验为 PNG 文件，保存时会把卡图烘焙并复制到版本目录 `moonspire/developer/card_art/<card_id>.png`，同时把该卡的 `artPath` 切换为对应的相对文件名，后续渲染直接引用该目录里的图片；再次更换卡图时会覆盖同名 PNG，清空/重置卡图时会删除对应的生成图片，避免留下旧文件；`物品图标` 打开可搜索、可滚动且不再按旧分类过滤的物品图标弹窗，弹窗顶部必须显示并独立渲染物品搜索框，关闭后不残留搜索框；`重置` 清空开发者覆盖图并恢复默认或无卡图。
- 卡面图片使用与卡牌卡图一致的本地 PNG 选择入口；选中后先引用原路径即时预览，保存时复制到版本目录 `moonspire/developer/card_faces/<face_id>.png`，并把卡面 `imagePath` 切换为对应的相对文件名。卡面底图渲染始终把整张 PNG 缩放到卡牌卡面尺寸，不裁剪源图；因此横图、竖图和任意大小源图都会完整填满当前卡面。卡面页不再依赖手填图片路径作为主要入口。
- 卡面图片保存后必须让 `imagePath` 指向 `moonspire/developer/card_faces/` 下的相对 PNG 文件名，并刷新该目标文件的动态贴图缓存；否则覆盖同名卡面 PNG 后可能继续显示旧贴图或回退到默认卡面。本地 PNG 选择后要单独记录本次选择的源文件直到保存完成，保存时优先从该源文件复制到目标 PNG，并在复制失败时用翻译 key 显示具体失败原因，不能静默吞掉导致界面看起来已保存但文件未落盘。默认卡面底图仍由源码资源 `card_base.png` 提供，默认卡面 id 保持 `default`，默认卡面的当前排布坐标以 `card_faces.json` 的 `default` 条目为准，当前默认名称区域为 `x:28 y:11 width:72 height:13`。
- 卡面页的费用、名称、卡图、类型和描述区域在预览图上直接编辑：点击区域选中，拖动移动；滚轮同时调整宽高，Shift + 滚轮调整宽度，Ctrl + 滚轮调整高度；Tab/Shift+Tab 切换区域，方向键移动区域，Ctrl + 方向键调整宽高。区域数据仍保存为 `DeveloperCardFace.Area` 的 `x`、`y`、`width`、`height`。卡面预览图上的区域文字按各自区域水平和垂直居中绘制，用于直接预览实际排版效果；费用区域固定显示数字 `1`，便于观察费用槽对齐效果。
- 卡面页底部按钮栏除通用 `保存`、`完成` 外，还提供卡面页专用的 `新建`、`删除`、`重置`、`应用`。`删除` 必须使用 `MoonSpireModalLayer` 顶层确认弹窗，并在确认或取消关闭时清理底层 hover、拖拽和滚动条抓取状态。删除非默认卡面后，所有使用该卡面的卡牌回退到默认卡面并选回默认卡面；删除默认卡面时恢复模组内置默认卡面。
- `应用` 会打开独立全屏卡面应用选择器，而不是 modal 弹窗。该界面继承 `NoBlurScreen`，顶部是搜索框，中间是卡牌列表，底部固定显示 `选中所有`、`取消所有`、`重置`、`应用`、`完成` 五个按钮。卡牌列表必须限制在屏幕边界内，并为搜索框、滚动条、第一排/最后一排卡牌完整显示和底部按钮预留明确空间；按钮文字保持居中，长卡名在卡牌下方按列宽缩放显示。
- 卡面应用选择器中，点击卡牌切换选中状态，再次点击同一卡牌取消选中；只有这种开发者可选择场景中的已选卡牌绘制蓝色边框。该蓝色边框不代表战斗目标、手牌选中或普通卡牌 hover 状态，不能扩散到非选择器的战斗 UI。
- 卡面应用规则：点击 `应用` 后，所有选中卡牌改用当前卡面；原本使用当前卡面但这次未选中的卡牌回退到默认卡面；若卡牌原本使用其它卡面而这次被选中，则取消其它卡面对该卡的应用并改为当前卡面。默认卡面等价于未特殊应用，不应为回退到默认的卡牌保留多余的 `faceId: default` 覆盖记录。
- 卡牌渲染卡面时必须优先按当前开发者数据中的规范化卡牌 id 解析最新卡面应用，其次回退到基础卡定义的卡面，最后才回退到 `CardInstance.faceId`。这样卡组、手牌、抽牌堆和弃牌堆中已经存在的旧 `CardInstance` 不需要重新生成，也能在卡面应用保存后显示最新卡面；开发者中心和卡面应用选择器必须把当前会话中尚未保存的 `DeveloperData` 显式传给卡牌渲染，保证点击 `应用` 后预览即时变化；不要只依赖实例里保存时的 `faceId` 或全局已保存数据。
- 渲染开发者卡牌实例时，如果 `CardInstance` 同时有注册 id 和 `developerCardId`，必须优先使用 `developerCardId` 匹配当前会话里的 `DeveloperCardDefinition`；否则 `custom_`/覆盖卡牌会被基础注册 id 抢先匹配，导致卡面应用后开发者中心预览仍显示默认卡面。
- 卡牌效果区通过 `+` 打开可搜索的效果列表，弹窗必须限制在开发者中心可视区域内并避开底部按钮栏，且作为最上层内容绘制，用实心暗色面板盖住并吞掉底层输入框、列表和卡牌交互；点击弹窗顶部搜索框必须把焦点交给该搜索框并吞掉底层点击，输入内容要立即筛选效果列表；切换卡牌、页签或打开其它弹窗时应关闭效果列表。当前提供造成伤害、获得格挡和施加流血。每条效果行左侧 `-` 会移除该效果，中间显示效果名称，右侧只在该效果需要目标且数值有效时显示可点击的 `目标` 按钮，否则显示灰色目标文本；点击 `目标` 会打开同款可搜索的目标列表选择器，列表包含自身、单体敌方、单体友方、全体敌方、全体友方、全体单位、自身以外的所有单位和随机敌方，选中后回填当前效果行。目标选择器与效果列表一样必须作为最上层内容绘制、吞掉底层输入；点击其顶部搜索框必须聚焦搜索框并立即筛选目标列表；内容溢出时支持连续滚动和滚动条拖拽。数值输入框直接编辑单次效果数值，次数输入框编辑该效果重复次数，次数默认为 1。只有带数值的效果能编辑次数。删除后数值和次数输入框必须同步移除。卡牌被切换选中、创建或恢复时，这组输入框要按当前卡牌效果列表重新生成，不能只保留上一次选中卡牌留下的控件。卡牌效果区内容相对边框内缩显示，并且卡牌效果区、效果选择弹窗和目标选择弹窗都要在内容溢出时支持连续滚动和滚动条拖拽，不允许盖住底部按钮栏。效果列表会转为卡牌统一效果列表、目标数据和次数数据。
- 卡牌效果列表还包含抽牌、获得费用、消耗、固有、保留、虚无、消耗手牌和丢弃手牌等统一效果；抽牌与获得费用允许修改目标，固有、保留、虚无和消耗属于无数值、无目标、无次数的关键词效果，因此效果行不显示数值框、次数框或目标按钮。
- 开发者中心使用左侧固定页签栏、中间固定表单区、右侧固定列表/预览区和底部固定按钮栏。左侧页签按钮必须按侧栏面板宽度水平居中；左侧栏底部还有全局 `删除所有改动` 按钮，按侧栏面板水平居中，点击后必须打开顶层确认弹窗。确认后会立即保存空开发者数据、删除生成图片并重置战斗 UI 布局覆盖，不再要求额外点击 `保存`。每个输入框上方必须绘制固定字段标签，不能只依赖占位文本说明字段含义。
- 开发者中心保存后重新接回同一界面时必须保留当前页签与对应选中状态，不得把卡面页等当前工作页强制跳回卡牌页；卡牌页保存后必须保留右侧选中的同一张卡，哪怕数据刷新后重新构建界面也一样；怪物页还必须保留右侧怪物列表的滚动位置。
- Moon Spire UI 的按钮文字默认保持水平居中和垂直居中；除非某个特殊按钮明确要求图标或文本左对齐，否则 `MoonSpireTextureButton` 和手绘按钮都必须把文字画在按钮中心，长文本需要在按钮内裁剪或缩放后仍保持居中。
- Moon Spire 纹理按钮必须在渲染时用当前鼠标坐标做实时命中判断，并在命中时绘制 `button_hovered.png`；不要只依赖可能未及时更新的旧 hover/focus 状态。新增或改动开发者界面按钮后，必须确认鼠标指向按钮时贴图会切换到高亮态。
- 开发者中心中没有注册成 widget 的手绘按钮，例如本地图片、物品图标、重置、卡面区域选择、效果加减和弹窗按钮，也必须使用当前鼠标坐标绘制高亮贴图，不能只修 `MoonSpireTextureButton`。
- 卡牌、卡面、怪物和战斗 UI 四个页签使用各自独立的布局规则：卡牌页包含中间表单、卡片预览、物品选择弹窗和卡组式小卡牌网格；卡面页包含表单、卡面预览和卡面列表；怪物页包含表单和怪物列表；战斗 UI 页只显示布局说明入口，不显示无关输入框。
- 怪物页的卡组字段不再使用可手填的卡牌 id 输入框，而是显示 `打开卡组` 按钮和当前有效卡组张数。未保存覆盖时，怪物页和怪物卡组界面读取该实体的默认战斗数据与实际默认卡组规则作为显示基础，外侧张数也显示该实体当前默认卡组数量；已有覆盖只替换对应字段，卡组只在用户实际编辑后写成覆盖。怪物页还显示 `打开奖励池` 按钮和当前有效奖励池张数；未自定义时奖励池显示为当前有效卡组的去重结果，自定义后显示保存的去重奖励池。点击后打开独立的怪物卡组/奖励池界面：卡组主界面只允许同时选择一张卡，底部固定显示 `添加`、`删除`、`复制`、`重置`、`完成`；奖励池主界面不显示 `复制`，并在添加时去重。`添加` 会进入全卡牌列表界面，该界面顶部有搜索框、允许同时选择多张卡，底部固定显示 `添加` 和 `取消`。当前卡组、当前奖励池和添加列表默认按卡牌显示名称排序，选择、删除和复制仍映射回真实条目。确认添加后把所选卡牌追加到当前列表并返回主界面，取消则不改动并返回主界面。两个界面都继承 `NoBlurScreen`，使用箱子式暗色背景、可拖动滚动条和卡牌蓝色选择边框。
- 怪物页底部提供 `重置` 和 `删除`。`重置` 将当前怪物覆盖恢复为上一次保存状态；`删除` 必须通过顶层 modal 确认，确认后移除当前怪物覆盖，使该怪物的血量、费用、速度和卡组回到默认状态。
- 开发者中心底部按钮栏中，`保存` 和 `完成` 是所有页签共用的固定按钮，其余操作按钮必须按页签单独创建。开发者中心主界面、卡面应用选择器和怪物卡组界面的底部按钮统一使用 64x20 尺寸和 8 像素间距；左侧页签按钮使用 90x20 尺寸，左侧全局操作按钮使用 100x20 尺寸，二者都按侧栏面板水平居中。页签专用按钮按可用区域居中，通用 `保存`、`完成` 靠右成组显示。卡牌页底部显示 `新建`、`删除`、`重置`、`筛选` 和 `给予`；`给予` 会把当前选中卡牌加入当前玩家卡组。卡牌库展开/收起使用卡牌列表左侧的小按钮，展开显示 `《`，收起显示 `》`，不再占用底部按钮栏；该按钮使用独立 `card_list_toggle.png`，其上下左右中心固定在卡牌库面板左边界与内容区垂直中线的交界处。`重置` 会把当前选中卡恢复到本次打开开发者中心后最近一次成功保存时的状态；若该卡是保存后新建且未保存的临时卡，则移除该卡并选回可用卡牌。删除必须先显示确认弹窗，确认弹窗必须作为最顶层 modal 绘制并吞掉底层输入，底层按钮不能盖住弹窗；确认后按规范化卡牌 id 删除开发者自定义/覆盖定义；卡牌页列表和删除后的选中回退必须以当前会话 `data` 叠加内置/转换基线卡为准，不能从已保存的开发者覆盖回读旧卡；内置怪物牌、转换牌和模组本体卡删除覆盖后会保持选中同一张卡并立即回退到默认定义，自定义卡删除后会选回当前筛选下的其它可用卡。删除弹窗确认或取消关闭时必须清理底层按钮 hover、拖拽和滚动条抓取状态，避免鼠标目标取消后仍残留瞄准/高亮。
- 右侧卡牌库筛选包含全部、模组本体、怪物、转换和自定义，不再带费用筛选。缩起时使用原本右侧宽度，展开时占满除左侧表单外的剩余宽度，高度不变；卡牌绘制区域必须给右侧滚动条预留命中宽度，卡牌和滚动条不得重叠。右侧区域较窄时，小卡牌要按可用列宽自动缩小显示，选中框必须完整留在裁剪区内，不能画出面板或滚动条区域；最后一行需要额外滚动空间避免被底部裁掉。
- 卡牌页没有选中卡牌时，左侧编辑字段、卡牌图片操作、效果编辑区和中间预览都不展示，也不能保留上一次选中卡牌的文字或预览残留；进入卡牌页默认也应保持无选中状态。点击右侧卡牌库空白处、搜索/筛选后当前卡牌不可见时，都按无选中状态处理，只保留右侧卡牌库和筛选信息。
- 右侧卡牌库第一行也必须有明确顶部内边距，且渲染、点击命中、首尾可见行计算和滚动内容高度必须使用同一套顶部/底部 padding，避免最顶上一张卡牌被搜索栏下方裁剪。
- 开发者中心的物品网格、卡牌网格、卡面列表、怪物列表以及其它可能超出可视范围的内容区必须遵守通用滚动条规则：内容放得下时不显示滚动条，内容溢出时允许滚轮滚动和直接拖动滚动条。
- 开发者中心的列表区域底部必须为按钮栏预留空间，列表点击范围和绘制范围都不能进入底部按钮栏。长卡牌 id、怪物 id、物品名和说明文字必须在各自列宽内截断或按可用高度换行，避免覆盖输入框、说明文字或按钮。
- 开发者数据主文件位于 `moonspire/developer/ui/developer_data.json`；保存时还会按功能同步写出 `card_faces/`、`cards/`、`monsters/` 子目录文件，卡牌最终 PNG 图写入 `moonspire/developer/card_art/`，卡面最终 PNG 图写入 `moonspire/developer/card_faces/`，并在保存后把卡牌数据中的 `artPath` 和卡面数据中的 `imagePath` 切换为对应的文件名。

## 手牌卡牌与开发者效果行

- 手牌中的小卡描述必须限制在卡牌描述区域内绘制，文字不能穿透到覆盖它的其它手牌上方。手牌悬停、拖拽指向目标、怪物已打出牌和卡牌预览使用完整卡牌尺寸的详细渲染参数，描述区域按可用高度显示更多行，同时卡图走不裁剪路径，避免缩放姿态下误裁物品图。
- 非本地已打出牌必须和怪物意图牌、玩家手牌一样按当前战斗双方速度、格挡和防护状态计算预览数值；即使该牌来自视觉事件、已经不在手牌或意图列表中，也要按视觉事件里的实际出牌者渲染伤害/格挡的增减颜色。
- 手牌选择效果打开时，`BattleScreen` 必须把该界面作为真正的顶层阻塞 modal：底层战斗条目、怪物意图、底部牌堆/费用/按钮、底层手牌、已打出的牌、出牌飞行动画、底板各区域文字、HUD 提示、牌堆提示、布局编辑器文字和基础 widgets 都不得绘制到遮罩上方，也不得响应出牌、结束回合、牌堆、目标、拖拽、右键和 ESC。代码应先跳过这些底层层级，再使用 `MoonSpireModalLayer.drawTopmostOverlay(...)` 绘制箱子式暗色阻塞层，最后只重绘 modal 自己拥有的候选手牌、选中手牌、标题和确认按钮。只有候选手牌和确认按钮可点击。
- 手牌选择层中的候选手牌按统一手牌布局规则排布：只要牌脱离手牌区域（被选中、拖拽或进入其它结算位置），剩余手牌立即按当前仍在手里的牌重新收紧；只要牌返回手牌区域，整组手牌立即重新铺开。被选中的手牌使用帧时间动画移动到确认按钮上方，多张牌并排显示；再次点击返回手牌区。候选牌悬停放大使用普通手牌悬停预览的缩放和屏幕边界钳制规则，不能超出屏幕，描述和关键词提示必须位于遮罩上方。未选够时顶部标题显示选择要求且确认按钮使用禁用纹理；选够后标题切换为确认文案，按钮切换为可用/悬停纹理。确认后选择层进入确认等待状态并播放结算动画：丢弃的牌从当前位置飞向弃牌堆，被消耗的牌留在原地播放消耗动画，当前实现为原地淡出，不得飞向消耗牌堆；等待期间仍绘制遮罩、候选/选中手牌、标题和禁用确认按钮，遮罩直到服务器快照清除本次选择流程后关闭。
- 开发者中心卡牌效果行的数值输入框前必须显示“数值”标签，次数输入框前必须显示“次数”标签；两个标签均使用翻译 key，不能写死中文或英文文本。新增标签后，目标按钮、数值框、次数框仍需在表单网格中从右向左排布，不能挤出滚动内容区域或影响弹窗命中。召唤效果行不显示目标按钮，目标列改为使用 `debug.moonspire.effect_entity` 提示的紧凑实体 id 输入框，并保存到 `DeveloperCardEffect.entityTypeId`；输入框宽度必须给效果名称保留足够空间，保证“召唤生物”完整可读，极窄宽度下优先缩小效果名称文字而不是只截出首字。

## 手牌重叠与效果行间距

- 普通战斗手牌处于扇形重叠状态时，小卡仍保留效果描述正文，并按自然堆叠顺序逐张完整绘制真实底图、卡图和文字；后绘制的前景手牌负责用自身 PNG 的真实透明像素覆盖下层内容，不得把底层文字或卡图按前景手牌切成大量可见片段，也不得用额外实色底片伪装遮挡。卡图在手牌、拖拽、放大预览、卡组和牌堆网格中都必须限制在卡图区域内渲染；稳定底部手牌只做整张完全离屏和描述文字完全离屏的保守剔除；悬停放大、拖拽详细卡和其它详细预览始终显示完整描述。
- 开发者中心卡牌效果行必须为左侧删除按钮和右侧目标按钮或召唤实体输入框保留最小安全间距。效果行从左到右依次为删除按钮、可截断的效果名、固定宽度目标按钮或实体 id 输入框、紧贴输入框的数值标签、窄数值输入框、紧贴输入框的次数标签和窄次数输入框；右侧数值/次数控件锚定在效果网格可视宽度内，不进入滚动条命中区，并优先压缩输入框宽度给效果名和目标列留空间。召唤实体 id 输入框使用比普通目标列更紧凑的宽度，但不得压缩到让效果名只显示单字；必要时效果名按比例缩小以保持“召唤生物”完整显示。效果输入框必须在效果列表自己的裁剪区域内单独渲染，滚动到半行可见时仍显示可见部分，不能因为未完整进入视口而整框消失。效果列表底部保留安全内边距，最后一行和 `+` 按钮不能贴住表单边框。
## 统一遮罩层

- 所有 Moon Spire 自绘的模态、弹窗、选择器、牌堆/卡组查看和战斗阻塞选择层，都必须使用箱子/物品栏式黑色透明遮罩。
- 遮罩统一通过 `MoonSpireUiTextures.drawOverlay(...)` 或 `MoonSpireModalLayer.drawTopmostOverlay(...)` 绘制，不允许使用灰色近乎不透明遮罩来掩盖层级问题，也不允许添加模糊效果。
- 绘制顺序必须是：底层界面正常作为背景渲染，随后绘制黑色透明遮罩，最后绘制当前模态内容。
- 遮罩打开时，底层战斗 UI 可以作为背景可见，但不得响应输入；底层文字、卡牌描述、hover 放大、已打出的牌、布局编辑文字和 tooltip 都不能绘制到遮罩之上。
- 战斗中的手牌选择层和牌堆查看层按 ESC 时打开暂停/设置菜单，不关闭当前选择或牌堆流程。
- 所有区域数字和底层 HUD 文字都属于遮罩下方内容，包括费用数字、抽牌/弃牌/消耗牌堆数量、角色生命/速度/意图数值、世界头顶伤害/效果数字、卡组/牌堆网格标题和数量、开发者布局文字等；这些内容不得使用高 z、`SEE_THROUGH` 字体模式或 see-through 渲染类型穿过遮罩。
- 费用数字、抽牌/弃牌徽章数字、消耗牌堆数字和结束回合按钮文字必须作为普通 GUI 文本绘制在底层；`MoonSpireUiTextures.drawOverlay(...)` 会先提交已经排队的底层 GUI 字体内容，再在高于底层 GUI 的 z 层绘制箱子式半透明遮罩并立刻提交遮罩本身，确保这些数字被遮罩压暗而不是穿透到遮罩上方。
- `MoonSpireUiTextures.drawOverlay(...)` 是唯一的整屏遮罩入口，绘制时必须压住之前已经提交的低层 GUI/世界文字，且遮罩提交完成后才能恢复后续 modal 内容绘制；如果某个数字或提示穿过遮罩，必须降低该元素层级、修正提交顺序或修正遮罩 z 层，不能通过提高遮罩不透明度解决。
- 模态打开期间，底层 hover 预览、放大卡牌、关键词提示、已打出的牌、飞牌动画和 tooltip 默认不绘制；只有当前模态拥有的卡牌、标题、按钮、预览和提示可以在遮罩上方绘制。
- 牌堆/卡组/弃牌/消耗牌网格也遵守同一规则：网格自己的标题、计数、卡牌和放大预览属于该网格模态内容；其背后的战斗 UI、其它网格或开发者底层文本不得越过遮罩显示。
- 战斗模态打开时，基础战斗条目整体不渲染；不要只隐藏血条文字或速度数字，因为基础条目的血量、速度、意图和效果数字都属于遮罩下方内容，必须避免重新穿层。
- 战斗与卡组相关界面不得在普通开发者模式下持续进行逐帧性能统计或日志采样；需要排查卡顿时只能临时开启明确的诊断入口，完成后应移除或关闭，避免诊断本身增加延迟。

## 战斗 UI 回归记录

- 本次性能修复后，战斗中的卡组、抽牌堆、弃牌堆和消耗堆不再从 `BattleSnapshot` 每 tick 携带完整卡牌列表，而是由 `BattleScreen` 按 `battleId + BattlePileSource + deckVersion + entityId` 请求内容包并缓存。弹窗刚打开时可以先显示遮罩、标题、数量和空网格，内容包到达后必须立即渲染真实卡牌；如果服务端回包版本新于当前快照，打开中的弹窗应直接显示该新版内容和回包数量。效果结算或手牌选择导致大快照暂时抑制时，只允许对当前打开的远端牌堆按客户端 tick 低频重验，不得重新引入每 tick 大快照携带完整列表。这只允许用于等待网络内容，不得把已收到的卡牌重新显示成占位或空卡。非战斗 `DeckScreen` 通过 `ClientCardState.version()` 判断收藏是否变化，稳定帧不得每帧全量比对所有卡牌；`CardGridPanel` 只能用调用方传入的内容 key/version 判断是否替换列表，不得重新引入整表签名或逐卡查找旧 hover 的稳定帧扫描。
- 打开的战斗牌堆弹窗在内容包到达后，不得每帧重复 `setCards(...)` 或重新构造网格数据；只有 `battleId`、`BattlePileSource` 或 `localDeckVersion` 变化时才能刷新内容。本地 PNG 卡图和自定义卡面允许异步补齐：首次遇到文件路径时不得在渲染路径同步解码或注册纹理，卡牌框、费用、名称、类型和描述必须先正常显示，卡图/自定义卡面在后台读取并由渲染线程分帧注册后逐步出现；这不是占位卡流程，也不得隐藏已经可绘制的真实卡牌文字。出牌展示卡属于底层战斗动画：它应在结算/本地等待期间保持中央短暂停留，结算结束后再飞向弃牌堆或淡出，不能因为快照刷新、牌堆弹窗刷新或模态层绘制而跳过停留或永久停在中央。
- 卡牌使用物品图标作为卡图时，`renderFakeItem` 后必须立即清理深度缓冲以免物品模型影响同一张卡的文字层或后续 GUI 层；常规卡图裁剪必须使用随当前 pose 变换后的裁剪矩形，不能用未变换的屏幕坐标裁剪缩放/平移中的卡牌。唯一例外是卡组/牌堆网格滚动运动帧中的居中物品模型卡图：这些帧可依赖外层滚动窗口裁剪以减少逐卡 scissor 状态切换，滚动停止后必须恢复逐卡物品卡图裁剪；自定义 PNG 卡图仍必须始终使用逐卡裁剪。卡组/牌堆网格中的小卡也必须保持真实卡图、卡框、费用、名称、类型和描述完整可见，不得为了性能跳过已有物品卡图、用普通物品 PNG 替代模型卡图或用占位卡替代真实卡牌。
- 卡组/牌堆网格中的重复卡牌应按卡牌内容复用轻量渲染准备结果：预热 key、描述换行缓存和开发者卡面查找不能包含每张复制品自己的 UUID，否则同样的卡多张会在打开卡组时重复构造文本与布局。网格小卡不得使用未经验证的整卡/卡图离屏纹理缓存，也不得绕开 `renderFakeItem` 改走手写批量物品模型路径；物品模型卡图必须回到原生 `renderFakeItem` 实时渲染，以保证卡图亮度、模型外观和显示稳定。本地 PNG 卡图、异步文件卡图和自定义卡面仍走原有实时渲染路径。
- 卡组/牌堆网格只需要绘制与当前裁剪窗口实际相交的卡牌内容；完全不可见的卡牌、以及卡牌局部露出但描述区域完全在裁剪窗口外的描述文本，可以不预热、不布局、不绘制。凡是玩家当前可见区域内的卡牌内容都必须保持完整，不得为了性能跳过可见的费用、名称、类型、描述或真实卡图；优化应优先复用重复内容的布局、行宽、文本测量和渲染准备结果。
- 卡组/牌堆网格卡顿的主要破局方向是减少重复文本工作：名称、类型、描述换行、描述最终布局和行宽应按卡牌内容 key 复用；如果日志显示 `textNameMs`、`textDescMs` 仍由字体提交本身主导，下一步应做可验证的文本层缓存或批量文字提交实验，不能通过隐藏可见描述、卡图或卡框来换取帧率。
- 已验证有效的卡组/牌堆优化包括：按内容 key 缓存名称/类型 `FormattedCharSequence`、文本宽度、描述换行和描述最终布局，按可见范围缓存当前帧 `CardValues` 与网格卡牌 pose，并避免稳定帧重复执行牌堆内容更新链。后续优化应继续围绕 `baseArtOtherMs`、`previewMs` 和战斗牌堆 `updateMs` 拆分定位，不能回退到整卡离屏缓存或跳过可见内容。
- 卡牌 UI 性能诊断只能通过显式 JVM 参数 `-Dmoonspire.perfDiag=true` 开启，日志前缀为 `[MoonSpirePerf]`；developer mode 不得自动启用逐帧诊断。默认关闭时，卡组、牌堆和战斗渲染热路径只允许保留布尔门控级别的诊断成本；开启后只记录超过阈值的服务端入战/快照、客户端接包、卡组/牌堆打开前 10 帧、卡组/牌堆滚轮或滚动条拖拽后的有限滚动诊断帧、PNG 解码/注册、物品图标渲染、描述缓存统计，以及网格内部 layout/warmup/overlay/hover/scissor/base/art/text/preview/title、滚动源、滚动偏移、`scrollMotion`、`itemArtClip`、可见索引范围、边缘文字跳过数和卡牌文字 cost/name/type/description 的细分耗时。
- 2026-05-09 的多人多怪相关 UI 修复中，不能把 `af7f788 部分内容修复调整` 当作正常显示基准；该提交后的卡组/牌堆和丢弃/消耗选择层已经存在渲染异常。需要对照正常旧表现时，应优先参考多人实现之前的 `d91b47d 模组基础内容补全`，再把多人多怪的数据结构和交互差异补回去。
- 已修好的战斗 UI 背景遮罩不能回退：`MoonSpireUiTextures.drawOverlay(...)` 需要先提交底层 GUI 文本，再以高于底层 GUI 的 z 层绘制并提交箱子式黑色半透明遮罩。修复遮罩穿透时不得隐藏底层战斗 UI，不得换成灰色近不透明遮罩，也不得通过加深遮罩掩盖层级错误。
- 卡组、牌堆、弃牌/消耗堆网格打开后必须立即渲染真实卡牌内容。不得重新引入会让大量卡牌显示为占位、空卡、只剩边框或只剩局部文字/图标的 placeholder/warmup 流程。若网格卡牌打开即缺失内容，优先检查遮罩后的 modal 内容 z 层、depth 状态、scissor 是否泄漏或被嵌套 `disableScissor()` 破坏，而不是先改卡牌数据。
- 卡组/牌堆网格在遮罩后绘制自己的卡牌、标题、滚动条、放大预览和关键词提示；这些内容属于 modal 自己，必须整体高于遮罩。遮罩下方的战斗 UI 背景继续可见但不可交互，且不能穿到网格内容上方。牌堆/卡组 modal 打开时，底层背景需要保留战斗条目、回合横幅和底部栏方向感信息；底部栏中的费用、牌堆入口和结束回合按钮仍可作为背景绘制但不可交互，手牌、怪物意图、飞牌、HUD tooltip 和其它被 modal 遮挡或不可交互的内容可以跳过渲染。
- 丢弃/消耗手牌选择层确认前必须绘制自己的遮罩、候选/选中手牌、标题和确认按钮；确认后客户端应立即进入本地确认等待态并关闭选择层，只保留底层战斗背景、隐藏所选手牌并播放对应飞牌/淡出动画，同时继续阻断底层输入，直到服务端快照清除、拒绝或改写该选择流程。
- 手牌选择层的悬停放大预览必须显示完整卡牌描述和关键词提示，关键词提示属于该 modal 自己的顶层内容，需在确认按钮等 modal 控件之后仍能绘制在遮罩上方。不要用卡牌预热、描述缓存或占位卡去绕过显示问题；如果确认后有长延迟，应优先检查本地确认等待态、服务端快照同步和动画流程，而不是让 modal 持续停留在禁用确认按钮状态。
- 结束回合按钮和顶部回合横幅是两个独立信息位。按钮内只允许四类文案：`结束第%s回合`、`敌方回合`、`你已死亡。`、`等待其他玩家结束回合`。顶部横幅只用于 `玩家回合` 和 `敌方回合` 的阶段切换提示，不显示 `回合结束`，也不得把顶部阶段文案随意塞进按钮。
- 结束回合按钮的文案和启用状态必须来自同一组状态判断。本地点击或按 Q 后可以临时记录等待服务端回包的禁用态，但该等待态必须绑定发起时的回合号；收到敌方回合、本地已结束回合或新的玩家回合快照时必须清除，避免下一回合已经显示 `结束第%s回合` 但按钮仍沿用上一回合的禁用状态。
- 结束回合按钮显示 `等待其他玩家结束回合` 时，必须同时满足本地玩家已结束或正在等待结束回合回包、且确实存在其它存活未结束玩家；单人战斗、其它玩家假死亡或其它玩家都已结束时，不得因为本地等待态短暂显示该文案。除 `你已死亡。` 和真正等待其它玩家这两种特殊禁用文案外，只有战斗状态本身可结束回合时才允许显示 `结束第%s回合`；打开卡组/抽牌堆/弃牌堆/消耗牌堆遮罩、弃牌/消耗手牌选择层，或本地玩家刚使用卡牌并等待权威快照/播放出牌动画时，底层结束回合按钮会因为输入锁而不能点击，但只要战斗状态本身仍处于玩家可行动阶段，文案和按钮底图仍保持可结束回合状态。若快照已经进入玩家回合但结算、手牌选择确认、本地结束回合回包、`ROUND_END` 过渡或本地已结束状态仍让按钮不可用，按钮仍显示 `敌方回合`，直到实际可点击后才显示 `结束第%s回合`；单纯残留的敌方出牌展示卡不再把结束回合按钮改成 `敌方回合` 或禁用底图。
- 大量卡牌打开卡组/牌堆卡顿、战斗开局卡顿仍是需要持续关注的性能问题；优化应优先做稳定缓存、减少重复布局/数据构造和移除临时诊断，而不能以隐藏真实卡牌内容、延迟显示真实卡面或破坏确认流程为代价。
- 当前有效的卡牌网格性能优化包括：按卡牌内容 key 缓存名称、类型、费用宽度、描述换行、描述布局和卡牌渲染计划；打开卡组、牌堆或弃牌/消耗堆时不得在首帧同步 warmup 大批可见卡牌，也不得先显示占位内容。真实物品卡图仍使用 `renderFakeItem`，并在一批可见卡图绘制完成后统一清理深度，避免每张卡重复清理。可见区域内的卡牌内容必须完整绘制，区域外内容可以跳过；网格小卡描述已经按描述框尺寸完成布局，绘制时不得为每张可见小卡再开一层描述 scissor；滚轮或滚动条拖拽造成实际滚动位移时，网格会在当前运动帧和极少量紧随帧中关闭逐卡物品卡图 scissor，改由外层滚动窗口裁剪，滚动停止或贴边拖拽不再改变偏移后恢复逐卡裁剪；诊断日志用 `scrollMotion` 和 `itemArtClip` 标记该状态。滚动边缘只允许跳过对应文字区域完全不在可视窗口内的费用、名称、类型或描述，不能跳过任何仍与可视窗口相交的文字区域；重复卡牌的 `contentKey` 构造缓存不得包含复制品 UUID，确保同内容多张卡复用同一份文本与布局准备；放大预览应复用网格已保存的 `contentKey`，不在悬停路径重复构造卡牌内容 key。
- 主战斗界面的性能优化可以使用有界的跨帧缓存复用同一张卡的 `contentKey` 与 `CardValues`，避免手牌小卡、描述可见性、悬停预览、拖拽牌和怪物意图重复构造内容 key 或重复计算伤害/格挡预览；`contentKey` 缓存必须按开发者数据版本、卡牌 UUID 与稳定的卡牌渲染字段 hash 失效，`CardValues` 缓存必须额外纳入战斗 ID、攻击者、怪物牌标记、当前指向目标、当前拖拽牌状态、卡牌渲染字段 hash，以及会影响预览数值的战斗单位速度、防御、假死亡和效果，不能因为无内容变化的快照心跳整批清空，也不能让目标、效果、费用或卡面变化后的显示过期。普通战斗手牌按自然堆叠顺序绘制真实小卡、`renderFakeItem` 物品卡图和文字；稳定底部手牌会先剔除整张完全离屏的非悬停手牌，再批量绘制其余手牌的底图和卡图，存在物品卡图时在这一批次后统一清理一次深度，然后每张卡只提交一次文字。底图和文字阶段必须复用同一份 `contentKey`，不能通过 `Map.getOrDefault(..., CardRenderHelper.contentKey(card))` 这类会无条件求值默认参数的写法重复构造热路径字符串。非悬停小手牌先用描述区域做低成本屏幕可见性粗筛；当描述区域贴近或越出屏幕边界时，再复用描述布局缓存计算真实描述文字包围盒，保证真实文字完全不可见时不提交。悬停放大牌、拖拽牌和手牌选择层仍完整绘制描述。普通手牌重叠不再按后续前景手牌遮挡关系切分卡图、费用、名称、类型或描述；这种分片 scissor 会在稳定战斗帧中放大 GUI 提交、字体提交和状态切换开销，导致 `bottomHandMs` 长期占用主渲染耗时，并可能造成渲染缺块。玩家实际可见的普通手牌卡图、费用、名称、类型、描述和交互详情必须保持不变，不得用占位图、离屏整卡缓存、跳过可见真实物品卡图、隐藏普通手牌可见文字或隐藏悬停/拖拽/预览详情换取帧率。`-Dmoonspire.perfDiag=true` 下会输出 `client.battle.render`，用于拆分主战斗界面的 snapshot、条目、意图、底部栏、飞牌、拖牌、怪物展示牌和其它段耗时；底部栏会继续拆分 energy/masks/piles/endTurn/flush/hand，手牌会继续拆分 base/baseFlush/pose/depth/text/contentKey/values/textVisibility/textSubmit/hover，并附带 `handCardOffscreenSkips`、`handDescOffscreenSkips` 以及本帧卡牌渲染统计，以确认卡顿是否仍集中在手牌小卡渲染、GUI 提交、物品卡图或文字绘制。后续排查仍未解释的耗时，应优先继续拆分 `baseArtOtherMs`、每帧对象分配、字体提交批次、`pushPose`/`popPose`、旋转缩放、scissor/depth 状态切换，以及是否能进一步合并卡牌文字提交。
- `-Dmoonspire.perfDiag=true` 的战斗同步诊断只记录 pending 抑制状态的首次变化和周期性摘要，不应在手牌选择或效果结算等待期间每 tick 输出一行日志，避免诊断 IO 自身造成卡顿或掩盖真正耗时。

## 手牌选择层

- 手牌选择层只允许候选手牌和确认按钮点击，禁止出牌、拖拽、结束回合、目标选择、牌堆入口、右键和底层 widgets。
- 未选够时标题显示选择要求，确认按钮禁用；选够后标题切换到确认文案，确认按钮启用。
- 选中手牌会从手牌区平滑移动到确认按钮上方，多张并排显示；再次点击返回手牌区。
- 手牌脱离或回到手牌区时，剩余手牌立即按统一手牌布局规则收紧或铺开。
- 确认后客户端立即关闭选择层，记录本地确认等待态，隐藏所选手牌并播放结算动画：丢弃牌飞向弃牌堆，消耗牌在当前位置淡出，不飞向消耗堆；等待服务器快照期间不再绘制选择层遮罩和禁用确认按钮，但底层战斗出牌、结束回合、目标选择、牌堆入口、hover 预览和拖拽仍被阻断。服务端继续回传同一个选择要求时，客户端必须视为旧 pending 尚未处理完成并保持本地确认等待，不得弹回确认界面；服务器快照清除选择流程后本地等待态结束，如果选择要求变化，则按新快照重建选择层。手牌选择层自己的卡牌、标题、按钮和预览在实际打开时必须在共享遮罩绘制完成后整体提升到 modal 内容层绘制，避免被遮罩的高 z 层或深度状态压住；底层战斗 UI 仍作为背景保留在遮罩下方，不得通过隐藏底层 UI 修复穿层。

## 牌堆与卡组预览

- 弃牌堆、消耗堆、卡组等网格界面的放大卡牌必须保留顶部和底部安全区；放大位置以原网格小卡中心为锚点，不得因为放大后超出屏幕边缘而向屏幕中间回弹，也不得被确认按钮遮挡。放大卡牌的关键词提示框仍需显示在屏幕内可见位置：优先贴近放大卡牌左右侧，若卡牌自身越界则只钳制提示框坐标，不反向移动放大卡牌。
- 卡组/牌堆网格的滚动条位置保持贴近原右侧位置；卡牌本体按一行可容纳 7 张的空间缩小计算，但实际网格仍只排 5 列。放大预览必须随网格卡牌同比例缩小，而不是继续使用完整 128x158 大卡尺寸。
- 牌堆/卡组放大预览属于当前网格模态自己的顶层内容，可以盖过该模态内的普通按钮；顶部标题和数量区域会作为同一模态的顶层遮罩最后重绘，因此放大卡牌和关键词提示进入顶部标题区时必须被该顶部遮罩压住。关闭预览时必须在动画低于可见阈值后立即清除整张预览，避免费用数字或卡面局部残留。`CardPreviewAnimation` 只统一非手牌卡牌预览的打开、缩回、可见阈值、当前动画卡面命中和动画本体尺寸，不决定最终预览位置；新动画首帧不推进并继续绘制原位卡，之后使用与手牌放大一致的推进节奏，动画卡只在可见阈值以上绘制，渲染使用浮点包围盒以避免像素抖动，关键词提示和边界判断仍使用整数包围盒。放大预览必须按原卡牌绘制路径等比缩放，不能在动画中切换到另一套卡面尺寸或重新排版文字/卡图。卡组/牌堆、怪物意图、奖励选牌和 JEI 各自继续使用自己的布局/边界规则；若当前列表、页面、意图或配方不再包含动画 key，必须立即清空旧动画，缩回完成的同一帧也必须清空并恢复原位卡。制卡台右侧固定预览不使用该动画。手牌和手牌选择层继续使用 `BattleScreen` 的 `HandCardAnimation`。
- 卡牌网格应缓存卡牌列表、布局结果、当前可见索引范围和列表建立时的 `contentKey`；打开卡组/牌堆时不使用会隐藏真实卡牌内容的占位卡面，必须保持完整卡牌渲染、悬停放大和关键词提示可用，同时通过缓存战斗卡组列表、网格布局、描述布局和渲染计划减少大列表打开与滚动时的卡顿。卡组/牌堆网格自己的卡牌、滚动条、标题和放大预览必须在共享遮罩绘制完成后整体提升到 modal 内容层绘制，遮罩下方的战斗 UI 背景保持可见但不得穿到网格内容上方。开发者中心、卡面应用选择器和怪物卡组列表应缓存筛选结果，减少打开大列表时的卡顿。`CardRenderHelper` 的帧级上下文只缓存开发者数据和卡面查找这类轻量、稳定的结果；文件贴图路径、物品图标和描述换行可以使用轻量缓存，但不得在热路径反复访问文件系统或构造高成本卡牌签名，避免大量卡牌界面越优化越卡。
- K 键打开的玩家卡组界面复用 `CardGridPanel` 的箱子式暗色遮罩、标题计数、滚动条和放大预览规则。非战斗 `DeckScreen` 只在没有其它屏幕时打开，并可再次按 K 关闭；战斗中只允许在基础 `BattleScreen` 且没有其它 modal/编辑器时打开自己的当前战斗卡组，打开后再次按 K 关闭该卡组界面。战斗中由 K 打开的卡组网格每帧从最新战斗快照刷新手牌、抽牌堆和弃牌堆内容；抽牌堆、弃牌堆、消耗堆和右键查看实体战斗牌库等其它网格弹窗不响应 K 关闭。
## 本次战斗 UI 布局补充

- `player_entry` 和 `monster_entry` 现在表示左右两侧的条目区起点与条目宽度。条目主体使用固定 42 像素高度，战斗状态图标绘制在主体下方的独立状态带中，条目步进会同时包含主体、状态带和条目间距，避免上一条目的状态被下一条目覆盖；条目区宽度跟随该基础条目的宽度，条目区底部默认延伸到消耗堆上方 8 像素处。
- 条目数量超过可见高度时，鼠标位于对应条目区内可滚轮滚动。该滚动不显示滚动条，点击、右键查看和卡牌指向命中都会使用同一套滚动偏移与裁剪区域，避免看得到但点不到或点到隐藏条目。
- 条目区只显示存活战斗单位。假死亡单位的条目会从列表中移除，世界中的选中轮廓、头顶血条、意图、状态、格挡动画与伤害数字也会停止显示；拖卡与点击选中不会再命中假死亡单位。
- 战斗中的假死亡单位仍先完整播放死亡倒下动画；倒下动画结束后，客户端会在实体碰撞盒范围内播放一次原版风格白色消散粒子，并在短暂消散阶段后隐藏该实体模型。假死亡隐藏只影响战斗中的世界视觉，不能继续保留红色死亡姿态、临时手持物、受击闪烁、冲刺偏移或自爆白闪残留。
- 指向/拖卡高亮优先于已选中高亮。拖卡指向某个目标时不会再同时保留旧选中怪物的黄色高亮，避免“当前指向目标”和第一只怪物一起高亮；拖卡指向条目时只使用当前条目/世界目标高亮，不再额外绘制旧目标区域大框，条目高亮范围必须覆盖战斗状态图标区域。
- 牌堆查看和手牌选择等 modal 关闭前后，底层战斗 UI 会作为背景继续绘制在标准箱子式黑色半透明遮罩下，避免退出丢牌/消耗牌界面时出现短暂空屏。
- 多怪物意图 UI 会根据当前指向或选中的怪物切换数据来源，意图小卡、放大预览和条目旁意图汇总都读取对应怪物自己的意图列表。进入敌方回合或其它服务端清空目标选择的时刻，客户端不会用上一只查看怪物维持意图显示；只有当前鼠标指向存活怪物/怪物条目，或服务端快照仍有选中怪物时，意图区才会显示。鼠标位于当前意图小卡或放大预览上时，意图区会锁定当前意图所属单位，避免同一鼠标位置同时命中背后生物后导致意图卡闪烁。
- 手牌拖拽、预览和打出后的飞行展示使用同一套按目标战斗状态计算的数值。玩家放开手牌出牌时，飞行展示牌会保存出牌瞬间的 `CardRenderData`，因此相位、凝视、守护等目标状态造成的数值变化不会在拖拽预览与打出展示之间不一致。
- 右键查看参战实体牌库的 `ENTITY_DECK` 覆盖层使用被查看实体自己的 `deckVersion` 和战斗牌库数量；缓存 key 必须包含 `entityId`，等待新版牌库内容包时保留当前卡片列表，只更新数量提示；若新版内容包先于对应快照到达，弹窗应直接切到新版内容，避免怪物出牌、抽牌、弃牌、消耗或生成箭时弹窗闪烁或停留旧内容。
- 世界战斗视觉中，近战直接伤害会先把攻击者移动到目标面前并停顿到命中反馈后再返回；玩家和怪物的近战冲刺、受击击退等战斗位移都由服务端视觉事件同步起点/目标点或初始击退速度，客户端在渲染前通过 `ClientBattleState.VisualState` 叠加平滑视觉偏移并按 tick 推进行走腿部摆动，服务端只在视觉结束或安全落点变化时做必要最终校正，不能用每 tick 玩家传送或其它逐 tick 强制位置覆盖来制作类似动画。`BattleVisualEvent.lookTarget` 会把本次卡牌视觉应朝向的目标眼位同步到客户端，`ClientEvents.playVisualEvents()` 在视觉事件消费当帧让攻击者用原版 `lookAt(...)` 转向并同步头部俯仰、头部 yaw 与身体 yaw，确保远程拉弓/装弩和近战冲刺开始时就看向目标；受击反馈统一通过原版 `LivingEntity.animateHurt(...)` 触发，玩家保留原版受击方向。骷髅家族的弓类远程视觉由战斗视觉状态控制临时主手弓、同一个 `ItemStack` 使用引用、使用时间和客户端 aggressive 状态，让原版 `SkeletonModel` 与弓物品模型自己进入持弓抬手和弓身拉开姿态，视觉后恢复。
- 开发者中心的删除确认属于 modal 顶层内容：基础界面先正常绘制，确认弹窗打开后跳过底层 widgets，使用 `MoonSpireModalLayer.drawTopmostOverlay(...)` 绘制箱子式黑色半透明遮罩，再在高于遮罩的 z 层绘制确认面板、文案和删除/取消按钮。渲染和点击必须共用同一组按钮 bounds；确认弹窗打开时 ESC 关闭弹窗并吞掉输入，底层按钮、滚动条、拖拽和选择器都不能被误触。

## 世界战斗动画补充

- 世界战斗动画会把服务端权威战斗结算和客户端视觉表现分层处理。三叉戟投掷和引雷投掷由服务端发送 `BattleVisualEvent.AnimationType.TRIDENT_THROW` / `CHANNELING_TRIDENT_THROW`，并在同一个事件中同步投射物起点、命中点、整段时长和 `projectileStack`；客户端临时替换主手三叉戟并表现蓄力，飞行阶段由 `ClientBattleState.ProjectileVisual` 在本地按 `age + partialTick` 连续插值渲染 render-only `ThrownTrident`，不把投射物加入世界实体列表，不依赖服务端逐 tick `setPos` 同步。溺尸在这些视觉期间会临时设置主手三叉戟和 `aggressive=true`，并跳过通用使用物品姿态，让原版 `DrownedModel` 自己进入举矛蓄力动作。
- 弓箭、光灵箭、弩箭、三叉戟和药水这类战斗投射物统一使用客户端本地投射物视觉链路：服务端只负责同步轨迹、结算时机、命中特效和最终规则效果；客户端在 `BattleWorldOverlay.renderLevel(...)` 的实体后渲染阶段用原版投射物 renderer 绘制 render-only 实体。后续新增世界战斗投射物不得通过服务端生成实体并逐 tick 移动来表现飞行，必须复用 `BattleVisualEvent.animationStart` / `animationStrike` / `animationTicks` / `projectileStack` 这条 partial-tick 渲染路径，避免网络同步造成投射物卡顿。
- 引雷投掷命中时，服务端在命中点生成 visual-only 闪电实体并播放雷击反馈；该闪电不依赖天气，不点火，不造成额外原版伤害。激流冲刺使用 `RIPTIDE_RUSH` 事件，客户端先播放蓄力，到冲刺阶段才进入原版 `Pose.SPIN_ATTACK`、spin flag 和激流纹理视觉；蓄力阶段仍以脚底坐标站在原地，释放冲刺阶段按攻击者包围盒半高把渲染路径切到身体中线，避免横向旋转模型贴地冲刺。玩家战斗视觉只写入用于渲染的 spin 字段和姿态，不调用 `startAutoSpinAttack(...)`，相机锚点在进入战斗时固定眼高，避免激流姿态改变本地玩家眼高后造成相机抖动；溺尸通过战斗专用渲染层复用原版激流纹理、模型层、旋转和缩放公式。激流视觉状态会保存并恢复原有 spin 字段、物品和姿态，服务端仍用战斗锁控制真实位置，避免为了视觉冲刺逐 tick 传送玩家，也不使用原版激流碰撞或伤害。
- 劫掠兽顶角头槌、碾压冲锋和践踏压迫的世界攻击视觉使用 `BattleVisualEvent.AnimationType.RAVAGER_HEAD_RAM`。服务端仍只同步冲刺起点、命中点和结算时机；客户端在 `ClientBattleState.VisualState` 中按 tick 计算原版 10 tick 攻击窗口，并由 `ClientEvents` 临时写入客户端 `Ravager.attackTick`，让原版 `RavagerModel` 负责头部伸缩。其它实体使用这些牌时不写入 `attackTick`，只走普通近战冲刺。
- 药水投掷和饮用不新增 PNG 资源，全部复用原版药水物品模型、`ThrownPotion` 实体和破裂粒子。`POTION_THROW` 事件会让客户端临时主手显示对应颜色的喷溅药水并在开始帧挥手，飞行阶段由客户端本地 `ProjectileVisual` 渲染 visual-only 药水实体，命中点由服务端播放原版药水破裂/溅落反馈后再结算 Moon Spire 卡牌效果；视觉药水不进入世界碰撞，不会给周围实体套原版药水效果。`POTION_DRINK` 事件会让玩家和其它 Humanoid 维持原版使用物品姿态 32 tick；女巫额外同步 `Witch#setUsingItem(true)`，让女巫模型进入原版饮药状态，结束后恢复进入视觉前的主手和饮药布尔状态。
