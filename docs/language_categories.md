# 语言文件分类

`src/main/resources/assets/moonspire/lang/*.json` 按语义分类维护。每一类之间保留一个空行；分类内部默认保留既有相对顺序，除非同一功能的小组需要放在一起。

新增、移动或删除语言键前，先阅读本文件。新增内容应放入下列匹配分类；如果出现新的前缀或现有分类无法准确表达用途，先更新本文件的分类说明，再修改语言文件。

新增语言键前，先搜索是否已有同类、同语义且占位符结构兼容的键。能够复用时优先复用现有键；只有玩家可见语义不同、上下文会造成误解，或 `%s` 等参数数量/顺序不兼容时才新增键。

## 分类顺序

1. 基础注册与按键
   - `block.*`
   - `item.*`
   - `itemGroup.*`
   - `key.*`

2. 普通界面与 HUD
   - `screen.*`
   - `hud.*`
   - 不包含开发者中心专用界面键。

3. 战斗阶段
   - `battle_phase.*`

4. 卡牌类型与卡牌效果文本
   - `card.moonspire.type.*`
   - `card.moonspire.effect.*`

5. 卡牌关键词
   - `keyword.*`

6. 状态效果
   - `effect.*`

7. 卡牌名称与卡牌描述
   - 其他 `card.*`
   - 例如怪物卡名称、未知卡名称、转换卡描述。

8. 玩家消息与命令反馈
   - `message.*`
   - `command.*`

9. 开发者工具
   - `screen.moonspire.developer*`
   - `debug.*`

10. 其他
   - 暂时无法归入以上分类的键。使用本分类时，应优先考虑是否需要新增更明确的分类。

## 维护规则

- `zh_cn.json` 与 `en_us.json` 使用同一套分类顺序和空行分隔方式。
- 同一语言键在不同语言文件中应尽量放在相同分类位置。
- 新增前优先复用同类语言键；新增后应确认没有产生可被已有键替代的重复键。
- 语言文件仍必须是合法 JSON；不要加入注释。
- 修改语言文件时，同时检查是否存在已经没有有效代码路径或资源引用的翻译键，并按需删除。
