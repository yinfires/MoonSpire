# Moon Spire 开发者功能说明

## 入口

- 需要在 `config/moonspire/client.json` 中开启 `developerMode`。
- 在世界内还需要 2 级管理员权限。
- `F8` 默认打开开发者中心；战斗中仍可切换战斗 UI 布局编辑器。该按键只有客户端启动时 `developerMode` 已开启才会注册到按键控制中，关闭开发者模式时不会显示问号占位项；运行时关闭开发者模式后按 F8 会静默无反应。

## 存储目录

游戏启动时就会创建版本目录下的开发者数据根目录：

`moonspire/developer/`

当前子目录：

- `moonspire/developer/ui/`：战斗 UI 布局覆盖。
- `moonspire/developer/card_faces/`：卡面模板数据和开发者卡面 PNG。
- `moonspire/developer/cards/`：全局自定义卡牌数据。
- `moonspire/developer/monsters/`：怪物覆盖配置。

主数据文件：

`moonspire/developer/ui/developer_data.json`

拆分数据文件：

- `moonspire/developer/card_faces/card_faces.json`
- `moonspire/developer/cards/cards.json`
- `moonspire/developer/monsters/monsters.json`

如果任一拆分数据文件存在，加载时以拆分文件为当前权威数据源：缺失的同类拆分文件会被视为空列表，而不会从旧的 `developer_data.json` 里把已删除内容复活。加载、打开开发者中心与保存都会清理无效的旧 `builtin_monster_*` 开发者卡牌覆盖，只保留当前代码仍注册的英文 canonical 怪物卡 id，避免拼音旧 id 卡在退出重进后重新写回磁盘；无效的非空怪物卡组覆盖被清空后，如果该实体有代码内置默认卡组，会自动取消覆盖并回退默认卡组。

## 开发者中心

开发者中心分四个页面：

- 卡牌：编辑全局自定义卡牌，支持物品图标或自定义图片。
- 卡面：编辑卡面模板图片和费用、名称、卡图、类型、描述区域。
- 怪物：编辑怪物血量、费用、速度、初始状态和卡组。
- 战斗 UI：继续使用现有布局编辑器。

## 卡牌规则

- 全局自定义卡牌默认不会自动进入玩家卡组。
- 管理员可以通过 `/moonspire give_card <card_id> [target]` 发卡给玩家。
- 怪物卡组可以直接引用全局自定义卡牌 id。
- 卡牌页选中卡牌后会显示 `复制` 按钮。点击后复制的是实际指令用的规范化卡牌 id：已有 `builtin_`、`item_`、`custom_` 前缀的 id 原样保留，其他开发者卡牌 id 会补成 `custom_<id>`。
- 开发者数据在保存和加载时会规范化并去重卡牌、卡面和怪物覆盖；怪物卡组中引用了已删除或无法解析的卡牌 id 时会被清理，但卡组内部允许保留重复卡牌和原始顺序。删除卡牌时，开发者中心会同步移除怪物卡组里的该卡引用并清理显示名缓存；保存后服务器还会清理在线玩家收藏和牌组中已经无法解析的自定义卡并重新同步玩家卡牌数据。

## 怪物规则

- 怪物搜索会同时显示译名和实体 id。
- 怪物覆盖会优先使用开发者数据中的血量、费用、速度、初始状态和卡组。
- 怪物卡组覆盖允许同一张卡出现多次；非空覆盖里的卡全部失效时，如果该怪物有默认卡组，保存/加载会取消这条无效卡组覆盖，战斗中也会回退到默认卡组。显式保存的空卡组仍表示开发者有意让该怪物没有战斗卡组。
- 怪物页的 `初始状态` 区域可以用 `+` 打开状态选择器，添加可用战斗状态；每条状态可以调整数值，也可以用 `-` 删除。不能为负层数的状态会把数值限制到不低于 0，数值无效或 0 层的状态不会保存为有效覆盖。
- 战斗开始创建怪物战斗单位时，服务端会从 `DeveloperDataManager.monsterOverride(...)` 读取该实体 id 的覆盖配置，先应用血量、费用和速度，再把 `initialEffects` 逐条转换成 `BattleEffectType` 并调用 `CombatantState.addEffect(...)`。之后仍会追加代码内置的默认初始状态，例如骷髅系的 `Abundant Arrows`。
- 怪物卡组编辑器中的 `复制` 会把当前选中的卡牌 id 在卡组列表中复制一份到下一行；它不是剪贴板复制。

## 卡面编辑

- 卡面页通过 `本地图片` 选择 PNG，选中后立即在卡面预览中显示。
- 保存时会把卡面 PNG 复制到 `moonspire/developer/card_faces/<face_id>.png`，并把卡面数据中的 `imagePath` 改为相对文件名。
- 任意尺寸卡面 PNG 都会在渲染时完整缩放到当前卡牌卡面尺寸，不做裁剪。
- 卡面预览中可以直接点选费用、名称、卡图、类型、描述区域并拖动移动；滚轮同时调整宽高，Shift + 滚轮调整宽度，Ctrl + 滚轮调整高度。
- Tab 和 Shift + Tab 可以切换当前区域，方向键移动区域，Ctrl + 方向键调整区域宽高。
- 卡面页底部增加 `删除`、`重置`、`应用` 三个按钮。`删除` 会打开顶层确认弹窗；删除非默认卡面后，使用该卡面的卡牌会回退到默认卡面并重新选中默认卡面。删除默认卡面时会恢复模组内置默认卡面。`重置` 会把当前卡面恢复到上一次保存时的状态。`应用` 会打开全屏卡面应用选择器。
- 卡面应用选择器顶部提供搜索栏，中间是可滚动的卡牌列表，底部提供 `选中所有`、`取消所有`、`重置`、`应用`、`完成`。卡牌可点击切换选中状态，再次点击同一卡牌会取消选中。
- 在卡面应用选择器中点击 `应用` 后，所有选中的卡牌会改用当前卡面；原本使用当前卡面但这次未被选中的卡牌会回退到默认卡面。默认卡面表示未特殊应用，不应在数据里保留一条等价的“默认应用”记录。
- 卡牌在卡组、手牌、抽牌堆和弃牌堆中渲染时，会按当前开发者数据中的规范化卡牌 id 解析最新卡面应用；已有的 `CardInstance` 即使仍保存旧 `faceId`，也会跟随保存后的卡面应用关系显示。开发者中心和卡面应用选择器会使用当前会话中尚未保存的数据渲染预览，因此点击 `应用` 后界面内的卡牌显示会立即变化。

## 备注

- 自定义卡图和卡面图片都建议通过开发者中心的本地图片入口选择。
- 物品图标会直接使用物品栏渲染图。
